# Шаблон — Новий профіль

Цей шаблон використовується, коли в [[Fitness — Профілі]] додається нова людина. Назви файлів мають бути у форматі `<Ім'я> — <Тип>.md`, окрім журналу: `Журнал — <Ім'я>.md`.

## Папка

```text
10-Projects/Fitness/<Name>/
```

## Обов'язкові файли

| Тип | Назва файлу | Призначення |
|---|---|---|
| Огляд | `<Ім'я> — Огляд.md` | дашборд профілю й поточний статус |
| План дій | `<Ім'я> — План дій.md` | одне місце з поточним фокусом і найближчими діями |
| Опитування | `<Ім'я> — Опитування.md` | базове опитування / onboarding |
| Параметри | `<Ім'я> — Параметри.md` | стислий структурований підсумок |
| Харчування | `<Ім'я> — Харчування.md` | меню, КБЖВ, правила корекції |
| Тренування | `<Ім'я> — Тренування.md` | програма, прогресія, обмеження |
| Сон | `<Ім'я> — Сон.md` | режим, якість, фактори ризику |
| Звички | `<Ім'я> — Звички.md` | куріння, вода, кроки, інші звички |
| Алкоголь | `<Ім'я> — Алкоголь.md` | факти, контекст, вплив на сон і форму |
| Журнал | `Журнал — <Ім'я>.md` | щоденні результати |
| Резервні звіти | `<Ім'я> — Резервні звіти.md` | ручні записи, якщо агент недоступний |

## Початковий статус

Новий профіль починається зі статусу `onboarding_required`.

Агент має:

1. створити каркаси файлів без вигаданих персональних даних;
2. провести базове опитування;
3. перенести підсумок у параметри;
4. створити перший план дій, харчування й тренування тільки після достатніх відповідей;
5. змінити статус на `active` після завершення onboarding.
