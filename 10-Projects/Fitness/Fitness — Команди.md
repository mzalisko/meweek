# Fitness — Команди

Єдиний контракт для агентів Claude і Codex у проєкті [[Fitness — Огляд|Fitness]].

Важливо: Codex CLI перехоплює всі повідомлення, які починаються з `/`, тому slash-команда `/fitness` у Codex не працює. Для Codex використовувати той самий зміст без слеша:

- `fitness`
- `fitness Микола`
- `fitness Марта`
- `fitness Микола smoke`
- `fitness <ім'я> імпортуй резервні звіти`
- `fitness Микола імпортуй резервні звіти`

У Claude команда доступна як `.claude/commands/fitness.md` і викликається через `/fitness ...`.

## Роутинг профілів

Джерело правди для профілів: [[Fitness — Профілі]].

Агент має:

1. прочитати [[Fitness — Контекст]] і [[Fitness — Профілі]];
2. визначити профіль за іменем у команді;
3. відкрити персональний огляд профілю;
4. працювати тільки з файлами цього профілю й спільними індексами;
5. якщо ім'я не вказане — показати доступні профілі й запропонувати вибір, не запускати щоденний режим автоматично;
6. якщо профіль має статус `onboarding_required` — спочатку провести onboarding і не вести щоденний режим до завершення базового опитування;
7. після завершення onboarding змінити статус профілю на `active` у [[Fitness — Профілі]] і персональному огляді.

## `fitness` / `/fitness`

Команда без імені не має змінювати дані. Агент показує доступні профілі з [[Fitness — Профілі]] і просить повторити команду з іменем, наприклад `fitness Микола` або `fitness Марта`.

## `fitness Микола` / `/fitness Микола`

Фокус на [[Микола — Огляд]].

Агент має:

1. прочитати [[Fitness — Контекст]] і [[Fitness — Профілі]];
2. перевірити, що Микола має статус `active`;
3. відкрити [[Микола — Огляд]];
4. якщо є щоденний звіт — додати запис у [[Журнал — Микола]];
5. якщо є алкогольний запис — додати його в [[Микола — Алкоголь]];
6. якщо є куріння/smoke — оновити [[Микола — Звички]];
7. якщо є дані про сон — оновити [[Микола — Сон]] або коротко зафіксувати сон у журналі;
8. оновити статистику, зведення і рекомендації тільки за достатніми даними;
9. не чіпати дані Марти.

Якщо користувач просить імпортувати резервні звіти, агент відкриває [[Микола — Резервні звіти]] і переносить записи зі статусом `новий` у відповідні файли.

## `fitness Марта` / `/fitness Марта`

Фокус на [[Марта — Огляд]].

Марта має статус `onboarding_required`.

Агент має:

1. прочитати [[Fitness — Контекст]] і [[Fitness — Профілі]];
2. відкрити [[Марта — Огляд]] і [[Марта — Опитування]];
3. якщо базове опитування не заповнене — не запускати щоденний режим;
4. провести режим первинного знайомства: зібрати базові дані, цілі, здоров'я, активність, харчування, сон, звички й алкоголь;
5. перенести стислий підсумок у [[Марта — Параметри]], [[Марта — Сон]], [[Марта — Звички]] і [[Марта — Алкоголь]] без зайвих приватних деталей;
6. створити стартові [[Марта — Харчування]] і [[Марта — Тренування]] тільки після достатніх відповідей;
7. після завершення onboarding змінити статус Марти на `active` у [[Fitness — Профілі]] і [[Марта — Огляд]];
8. після цього використовувати той самий щоденний механізм, що й для Миколи: [[Журнал — Марта]], [[Марта — Резервні звіти]], статистика і рекомендації.

## `fitness Микола smoke` / `/fitness Микола smoke`

Окремий режим для куріння Миколи.

Агент має:

1. відкрити [[Микола — Звички]];
2. оновити день без куріння або записати зрив;
3. якщо користувач дав тригер — додати його в детальний журнал;
4. не скидати весь прогрес через один зрив, а записати факт і наступну дію;
5. якщо дата не вказана — використовувати поточну дату сесії.

Приклади:

```text
fitness Микола smoke сьогодні 0 сигарет, тяга 3/10, тригер кава
```

```text
fitness Микола smoke був зрив 1 сигарета, тригер стрес, наступна дія прогулянка
```

## Майбутні режими

- `fitness Микола alcohol` — окреме внесення алкоголю, напоїв і коктейлів.
- `fitness Микола weight` — швидке внесення ваги.
- `fitness Микола workout` — швидке внесення тренування.
- `fitness <ім'я> sleep` — окреме внесення сну.
- `fitness <ім'я> імпортуй резервні звіти` — імпорт резервних записів для будь-якого активного профілю.

## `fitness <ім'я> імпортуй резервні звіти`

Режим для ситуації, коли агент був недоступний через ліміт. Працює для активного профілю; якщо профіль має статус `onboarding_required`, спочатку завершити onboarding.

Агент має:

1. визначити профіль через [[Fitness — Профілі]];
2. відкрити резервні звіти цього профілю: [[Микола — Резервні звіти]] або [[Марта — Резервні звіти]];
3. знайти рядки зі статусом `новий`;
4. перенести щоденні дані у журнал профілю;
5. окремо перенести алкоголь, сон і звички у відповідні персональні файли, якщо ці поля заповнені;
6. змінити статус на `імпортовано` або `уточнити`;
7. оновити [[Fitness — Контекст]] і зробити коміт.
