# План імплементації 2.2 — Панель слота (редагування + публікація)

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans. Steps use checkbox (`- [ ]`).

**Goal:** Права панель слота в адмінці: клік на рядок-телефон у гріді відкриває панель із ланцюгом номерів, статусами, прив'язаними месенджерами й поведінкою; менеджер може закріпити номер (pin/unpin), змінити режим повернення (Авто/Sticky) і політику вичерпання, увімкнути/вимкнути прив'язаний месенджер, та натиснути **Зберегти** → публікація уражених сайтів у DataBridge (аудит уже пишеться сервісами).

**Architecture:** Новий Livewire-компонент `SlotPanel` поряд із `ValuesGrid`. Грід при кліку на телефон-рядок діспатчить подію з ключем значення; `SlotPanel` вантажить `PhoneSlot` за `DataValue.key`+scope і рендерить праву панель (макет §панель). Дії використовують наявні сервіси: `FailoverEngine::pin/unpin/recompute` (вони ж пишуть аудит), прямі `PhoneSlot::update` для режиму/вичерпання, `DataValue` content для месенджерів. **Зберегти** публікує уражені сайти через `SitePayloadCompiler::publish` + `BridgePublisher::push` (як webhook у 1.2). Без staging — дії застосовуються одразу, Save лише пушить (preview/відкат — пізніше).

**Tech Stack:** Laravel, Livewire 3, Tailwind, PHPUnit + Livewire::test, Http::fake для bridge-push.

**Спека:** `DBManager — Дизайн.md` §4 (слоти/failover), §7 (панель слота). Макет: `.superpowers/brainstorm/2289-1781202386/content/grid-failover-v5.html` — секція `.panel` (рядки ~174–210): заголовок «Слот: …», гео-мітки, ланцюг номерів із grip/статусами/«Показувати цей» (pin), прив'язані месенджери (увімкнено/вимкнено), Поведінка (Повернення Авто/Sticky, Ручний режим зі «Зняти», «Якщо всі впали: Прибрати вивід»), кнопка «Зберегти → аудит + push».

**Сервіси (джерело правди API):** `App\Services\Failover\FailoverEngine` — `pin(PhoneSlot,NumberEntry)`, `unpin(PhoneSlot)`, `recompute(PhoneSlot)`, `sitesFor(PhoneSlot): Collection<Site>` (усі пишуть аудит). `App\Services\Publishing\SitePayloadCompiler::publish(Site): Publication`. `App\Services\Publishing\BridgePublisher::push(Publication): bool`. Моделі: `PhoneSlot` (return_mode auto|sticky, exhaustion_policy hide|last|emergency, pinned_number_entry_id, current_number_entry_id, dataValue), `NumberEntry` (priority, phoneNumber{status,e164}).

**Робоча директорія:** `C:/Dev/Meweek/10-Projects/DBManager/code` (bash). Тести: `docker compose run --rm core php artisan test`. Фронт: node:22 `npm run build` (з `MSYS_NO_PATHCONV=1` у Git Bash). Git — коміти `dbmanager(admin): ...`, шляхи під `code/core`, без push (крім фінального).

**Поза скоупом 2.2 (свідомо):** drag-перетягування пріоритетів резервів (поки кнопки ↑/↓ або без зміни порядку — порядок задається в моделі); staging/прев'ю змін і відкат пакета; додавання/видалення номерів і месенджерів (лише перемикання наявних). Ці — план 2.3.

```php
    use BuildsSlots;

    protected function setUp(): void
    {
        parent::setUp();
        $this->actingAs(User::factory()->create());
    }

    public function test_opening_slot_shows_number_chain(): void
    {
        $site = Site::factory()->create();
        [$slot, $entries] = $this->slotWithNumbers(['down', 'active']);
        $slot->dataValue->update(['key' => 'phone_ua_1', 'scope_type' => 'site', 'scope_id' => $site->id]);

            ->call('open', $slot->dataValue->id)
            ->assertSee('phone_ua_1')

            ->assertSet('open', true);
    }

    public function test_closed_by_default(): void
    {
        Livewire::test(SlotPanel::class)->assertSet('open', false);
    }
}
```

### Task 3: Pin / Unpin номера

- [ ] **Step 1: Тест** — `pin($entryId)` закріплює (slot.pinned_number_entry_id = entry) і показуваний стає цим номером; `unpin()` знімає. Перевірити через `FailoverEngine`/resolver і запис у `audit_logs` (action slot.pinned).
```php
public function test_pin_makes_number_current_and_audits(): void
{
    $site = Site::factory()->create();
    [$slot, $entries] = $this->slotWithNumbers(['active', 'active']);
    $slot->dataValue->update(['scope_type'=>'site','scope_id'=>$site->id]);

    Livewire::test(SlotPanel::class)->call('open', $slot->dataValue->id)
        ->call('pin', $entries[1]->id);

    $this->assertSame($entries[1]->id, $slot->fresh()->pinned_number_entry_id);
    $this->assertTrue(\App\Models\AuditLog::where('action','slot.pinned')->exists());
}
```

- [ ] **Step 1: Тест** — `save()` публікує всі сайти зі `sitesFor($slot)` у bridge (Http::fake; assert POST на ingest_url із domain ураженого сайта), показує флеш «збережено».
```php
public function test_save_publishes_affected_sites_to_bridge(): void
{
    config(['services.bridge.ingest_url'=>'https://bridge.local/api/internal/publish','services.bridge.publish_secret'=>'pub']);
    \Illuminate\Support\Facades\Http::fake(['*' => \Illuminate\Support\Facades\Http::response(['stored_version'=>1],200)]);
    $site = Site::factory()->create(['domain'=>'domen.ua']);
    app(\App\Services\Provisioning\SiteProvisioner::class)->issueToken($site);
    [$slot] = $this->slotWithNumbers(['active']);
    $slot->dataValue->update(['scope_type'=>'site','scope_id'=>$site->id]);

    Livewire::test(SlotPanel::class)->call('open', $slot->dataValue->id)->call('save');

    \Illuminate\Support\Facades\Http::assertSent(fn($r) => str_contains($r->url(),'internal/publish')
        && json_decode($r->body(),true)['domain']==='domen.ua');
}
```
