# План імплементації 2.8 — Сайти і групи

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development. Steps use checkbox (`- [ ]`).

**Goal:** Новий екран `/admin/sites` (Livewire `SitesManager`) — патерн «список + права панель» (як `AccessManager`/`SlotPanel`). Це **точка входу** в систему: керування групами і сайтами (CRUD з **архівуванням/soft-delete**), токенами сайта (видати/відкликати/ротувати, сирий токен показати раз), статусом зв'язку (остання звірка `last_seen_at`, версія публікації). З рядка сайта кнопка **«Керувати даними»** відкриває **наявний грід «Значення»**, вже прив'язаний до цього сайта — той самий вигляд і способи додавання номерів-резервів і месенджерів, що вже є. Активувати пункт сайдбару «Сайти і групи». RBAC + аудит на всі зміни.

**Чому цей план:** користувач очікував, що дані вводяться «для сайту», а грід «Значення» — глобальний редактор, лише відфільтрований по обраному сайту. Цей екран робить потік явним: **створити групу → додати сайти → з сайта зайти в його дані** (наявний грід). Див. [[DBManager — Дизайн]] §7 (сайдбар «Сайти і групи»), §4 (Site/SiteGroup/Token), §9 (токени: один сайт = один read-only токен; відкликання за секунду; ротація).

**Архітектура:**
- **Soft-delete:** додати `deleted_at` у `sites` і `site_groups` (міграція) + трейт `SoftDeletes` на моделях. Архівні приховані зі списків, перемикача сайтів і гріда (SoftDeletes виключає trashed автоматично); у `SitesManager` — тумблер «Показати архів» + «Відновити».
- **`SitesManager` (Livewire):** зліва групи секціями з вкладеними сайтами (домен, country_hint, бейдж зв'язку, лічильник значень); права панель — форма групи **АБО** форма сайта (взаємне виключення `panelMode: 'group'|'site'|null`).
- **Токени:** розширити `SiteProvisioner` методом `revokeToken(Site)`; ротація = revoke + issue у компоненті; сирий токен показати один раз через `visibleToken` (патерн `visiblePassword` з `AccessManager`).
- **Статус зв'язку:** останній невідкликаний `ApiToken.last_seen_at` (null → «ще не підключався»; свіже → онлайн) + `Publication::where('site_id',…)->max('version')`.
- **RBAC (v1):** маршрут і всі мутації — за `AccessControl::canManageAccess` (суперадмін), як `/admin/access`. Гранулярні права менеджерів на структуру — наступний крок (нотатка в self-review).
- **Архівування і дані:** архів сайта/групи **не чіпає значень** — вони лишаються, сайт лише прихований; відновлення повертає все. Архів групи **каскадно архівує її сайти** (відновлюване) з підтвердженням.

public function test_archive_group_cascades_to_sites_and_restore_brings_back(): void
{
    $this->actingAs($this->superadmin());
    $g = SiteGroup::factory()->create();
    $s = Site::factory()->for($g, 'group')->create();
    Livewire::test(SitesManager::class)->call('archiveGroup', $g->id);
    $this->assertSoftDeleted('site_groups', ['id' => $g->id]);
    $this->assertSoftDeleted('sites', ['id' => $s->id]);
    Livewire::test(SitesManager::class)->call('restoreGroup', $g->id);
    $this->assertDatabaseHas('sites', ['id' => $s->id, 'deleted_at' => null]);
}

### Task 4: Токени сайта (видати / відкликати / ротувати, показ раз) + статус зв'язку

- [ ] **Step 1: Тест** — issue показує сирий раз і зберігає лише хеш; revoke ставить `revoked_at`; rotate відкликає старий і видає новий; audit:
```php
public function test_issue_token_shows_raw_once_and_stores_hash(): void
{
    $this->actingAs($this->superadmin());
    $site = Site::factory()->create();
    $c = Livewire::test(SitesManager::class)->call('editSite', $site->id)->call('issueToken');
    $raw = $c->get('visibleToken');
    $this->assertNotEmpty($raw);
    $this->assertDatabaseHas('api_tokens', ['site_id' => $site->id, 'token_hash' => hash('sha256', $raw)]);
    $this->assertDatabaseMissing('api_tokens', ['token_hash' => $raw]); // сирий не зберігається
    $this->assertTrue(AuditLog::where('action', 'token.issued')->exists());
}

public function test_revoke_token_sets_revoked_at(): void
{
    $this->actingAs($this->superadmin());
    $site = Site::factory()->create();
    app(SiteProvisioner::class)->issueToken($site);
    Livewire::test(SitesManager::class)->call('editSite', $site->id)->call('revokeToken');
    $this->assertNull(app(SiteProvisioner::class)->activeTokenHash($site->fresh()));
}
```

- [ ] **Step 3: Реалізація**
