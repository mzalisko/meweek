# DBManager — Контекст

**Оновлено:** 2026-06-19
**Стан:** Впроваджено масове форматування телефонів, нормалізацію номерів та відкат масових змін. Усі тести проходять.

## Актуально
- Код проєкту: `10-Projects/DBManager/code/`.
- Додано масове встановлення формату телефону (`set_phone_format`) з валідацією.
- У таблицях прев'ю та значень телефони показуються без маски (сирі), а формат винесено в окрему колонку "Форматування", яка також підтримує редагування.
- Виправлено стилі таблиць (замінено dynamic-класи сітки на inline-стилі `style="grid-template-columns: ..."` для гарантованого горизонтального рендерингу).
- У таблиці значень відображається тільки основний номер; резервні номери винесено в окремий розділ "Зміна резервів" у бічній панелі.
- У масових операціях додано розділення на "Телефони (основні)" та "Резервні телефони" (phone_reserve) для точкової зміни.
- Додано панель скасування останніх 3 сесій масових змін у `/admin/bulk` (відкрита за замовчуванням `<details open>`) з локалізованими змінами.
- Впроваджено автоматичне очищення масок (нормалізація до сирого e164) при додаванні чи редагуванні номерів у `ValuesGrid` та `SlotPanel`.
- Покращено деталізацію в аудиті:
  - Додано підтримку відновлення (відкату) для всіх типів масових дій (`bulk.*`).
  - Виправлено відображення змін ключів для `bulk.replace_text` (раніше писало "Оновлено внутрішні параметри").
  - Реалізовано відображення створених значень для `value.created` (навіть якщо у логу немає кореневого ключа `content`).
