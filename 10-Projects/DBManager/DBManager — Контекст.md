# DBManager — Контекст

**Оновлено:** 2026-06-19
**Стан:** Реалізовано панель геосимуляції в адмінці WP плагіна (з гарантованими країнами UA, RO, RU, BY, KZ, PL), автоматичне виконання шорткодів у ACF полях, `the_title`, `the_excerpt`. Проведено модульний рефакторинг та російську локалізацію. Створено гайди з підключення та CRM-інтеграції. Тести плагіна проходять на 100%.

## Ключові зміни
- **Гарантовані країни симуляції:** Додано базовий набір країн (UA, RO, RU, BY, KZ, PL), який завжди доступний на панелі геосимуляції.
- **Гайди користувача та розробника:** Створено покроковий посібник підключення плагіна та посібник з інтеграції з CRM (інтеграція по API статусів та через прямі SQL запити).
- **Локалізація інтерфейсу:** Перекладено інтерфейс адмін-панелі плагіна на російську мову.
- **Тип ЦІНА:** Повна реалізація — ValueEditor, відображення в гріді, аудит-лог змін, SitePayloadCompiler, render-core (пріоритет точної країни → WORLD fallback).
- **Дозвіл `can_view_prices`:** Міграція на `user_site_access`/`user_site_group_access`, AccessControl, AccessManager, UI колонка «Ціни», фільтрація в ValuesGrid.
- **Локальні дані:** Кожен сайт зберігає та читає власні значення (`scope_type = 'site'`).
- **Тести:** Всі 236 тестів Core (706 assertions) та 37 тестів плагіна проходять успішно.

## Шлях коду та запуск
- Код: `10-Projects/DBManager/code/`
- Тести Core: `docker exec -i code-core-1 php artisan test`
- Тести плагіна: `docker compose run --rm plugin vendor/bin/phpunit`
