# Інструкції для AI-агентів (Meweek)

Цей файл — спільні правила для всіх агентів (Codex, Claude, Gemini та інших), що працюють у цій папці.

## Що це за середовище

`C:\Dev\Meweek` — це одночасно:

1. **Сховище Obsidian** — нотатки, документація, плани (українською).
2. **Робоче середовище проєктів** — код проєктів живе в `10-Проєкти/<Назва>/код/` або в окремих репозиторіях, на які посилаються нотатки.
3. **Git-репозиторій** — https://github.com/mzalisko/meweek

## Обов'язкові правила

1. **Мова**: всі `.md` файли, коміти, коментарі в документації — українською. Код і назви змінних — англійською.
2. **Структура**: нові нотатки створювати у правильному розділі (`00-Вхідні` для чернеток, `10-Проєкти/<Назва>/` для проєктних, `20-База знань` для постійних знань). Не створювати файли в корені сховища без потреби.
3. **Посилання**: пов'язувати нотатки через `[[wiki-посилання]]`. Кожна проєктна нотатка має посилатися на огляд свого проєкту.
4. **Шаблони**: для нових проєктів, ADR, зустрічей використовувати шаблони з `_Шаблони/`.
5. **Не чіпати**: `.obsidian/` (конфіг Obsidian) — редагувати лише на явне прохання.
6. **Git**: осмислені коміти українською у форматі `розділ: що зроблено` (напр. `nmdb: додано архітектурну схему`). Не пушити без прохання користувача.

## Безпека

- Жодних секретів (паролі, API-ключі, токени) у нотатках чи комітах. Секрети — лише в `.env` файлах, які в `.gitignore`.
- Проєкт NMDB має підвищені вимоги до безпеки — див. [[10-Проєкти/NMDB/NMDB — Безпека]].
