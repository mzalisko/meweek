# AI OS — практики Four Cs

Методологія побудови особистого «AI-середовища» (зі скріншотів) + актуальні практики Anthropic щодо навичок. Так побудовано це сховище.

## Four Cs — порядок побудови

| C | Що це | У нас |
|---|---|---|
| 1. Context | Хто ти, правила, нотатки («другий мозок») | Сховище Obsidian, `AGENTS.md`, розділи 00–90 |
| 2. Connections | Твої реальні дані | Конектори: GitHub, Gmail, Notion… ([[30-Агенти/Конектори і навички]]) |
| 3. Capabilities | Навички + агенти | `.claude/skills/` (new-project, save-context, weekly-review) |
| 4. Cadence | Працює саме, ритм | Заплановані задачі, weekly-review |

Порядок не форсується: «usage pulls you to the next C» — наступний рівень додається, коли його потребує реальне використання.

## CLAUDE.md = роутер (routing tree)

- Одна папка для всього; `CLAUDE.md` тримає **вказівники, а не знання**: «потрібно X → дивись там».
- **Індекс на кожному рівні**: корінь → `Старт.md`, розділ → `<Розділ>.md`, проєкт → `<Назва> — Огляд.md`. Агент іде одразу до джерела.
- Якщо щось не знаходиться за 2–3 кліки по індексах — це перша проблема, яку треба виправити.

## Два шари — одна система

- **Другий мозок (пам'ятає):** нотатки, рішення (ADR), транскрипти → розділи 00–90.
- **AI OS (діє):** навички, автоматизації, рутини → `.claude/`, заплановані задачі.

## Практики Anthropic для навичок (актуально на 2026)

1. **Progressive disclosure** — SKILL.md короткий; деталі — в окремих файлах поруч, які читаються на вимогу. Понад ~300 рядків — точно ділити.
2. **«Пушна» description** — у ній і що робить навичка, і конкретні тригери-фрази, бо моделі схильні недовикористовувати навички.
3. **Правило + чому** — замість КАПСУ пояснювати причину правила, щоб агент узагальнював на нові випадки.
4. **Ітерація з спостережень** — дивитись, як агент реально використовує навичку, і правити.
