# Claude Code — Рекурсивні субагенти

## Що це

З Claude Code v2.1.172 (10 червня 2026) субагенти можуть породжувати власних субагентів — до 5 рівнів рекурсії. Раніше агент міг лише сам виконувати задачі; тепер він може делегувати підзадачі іншим агентам, які своєю чергою можуть делегувати далі.

## Чому це важливо

Це переводить Claude Code від "один агент — одна задача" до справжніх ієрархічних та паралельних воркфлоу. Оркестратор розбиває задачу, субагенти виконують частини паралельно, результати агрегуються. Те, що раніше потребувало ручної оркестрації через кілька сесій, тепер відбувається в межах однієї.

## Приклад

```
Оркестратор: "Проведи weekly review сховища"
  ├── Субагент 1: "Перевір проєкт Fitness"
  │     └── Субагент 1.1: "Перевір відкриті задачі"
  ├── Субагент 2: "Перевір проєкт DBManager"
  └── Субагент 3: "Переглянь 00-Inbox"
```

## Як використати в Meweek

- **weekly-review**: запускати паралельні субагенти для перевірки кожного проєкту окремо.
- **daily digest**: субагенти для кожного джерела новин паралельно.
- **code review**: субагенти для різних аспектів (security, logic, style).

Виклик субагента в навичці або промпті — через [[Agent tool]] з `run_in_background: true` для паралельного виконання.

## Обмеження

- Максимум 5 рівнів вкладеності.
- Кожен субагент витрачає окремі токени — враховувати при плануванні.
- Після 15 червня 2026 — субагенти в `claude -p` тарифікуються за API-ставками окремо від підписки.
