# Claude Code — Dynamic Workflows

## Що це

Dynamic Workflows — функція Claude Code (research preview з Week 22, травень 2026), яка дозволяє одному скрипту оркеструвати десятки або сотні субагентів. Скрипт описує логіку паралелізації, Claude виконує агентів автоматично і повертає синтезований результат.

## Чому важливо

- Замінює ручне керування кількома агентами в складних задачах
- Дозволяє справжній паралелізм: агенти шукають, аналізують, верифікують одночасно
- Підтримує ієрархічне spawning до 5 рівнів вглибину
- Доступно на Claude API, Bedrock, Vertex AI, Microsoft Foundry

## Приклад

```
claude --workflow "Знайди всі баги в src/, кожен файл перевір окремим агентом, зведи звіт"
```

Або через API з `dynamic: true` параметром у запиті.

## Як можу використати

- **daily-ai-digest**: 5-7 пошукових агентів паралельно по різних джерелах, 1 агент-синтез
- **weekly-review**: паралельний огляд проєктів, потім зведення
- **fitness-аналіз**: кожен тренований аналізується окремим агентом
- **code audit**: кожен файл/модуль — окремий агент-рев'ювер

## Застереження

- Суттєво більше токенів, ніж звичайна сесія — починати зі scope-обмеженої задачі
- Research preview — API може змінитися
- Оптимальний для задач, де є природний fan-out (багато незалежних підзадач)
