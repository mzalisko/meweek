# Software 3.0 — концепція Karpathy

## Що це

Розширення тези "Software 2.0" (Karpathy, 2017) — концепція про те, *коли і чому* автоматизація через LLM є надійною і самовдосконалюваною.

**Ключовий принцип**: автоматизуй те, що людина може верифікувати. Якщо правильний результат можна перевірити автоматично (тест-сюіт, ігровий рахунок, формальний proof-чекер, метрика якості) — LLM здатен навчитися або бути промптованим виробляти цей результат надійно.

## Чому важливо

Дає **практичний критерій відбору задач для автоматизації**:
- До автоматизації питай: "чи можна автоматично перевірити результат?"
- Якщо так — автоматизація стійка, надійна, і може самовдосконалюватись через feedback loop.
- Якщо ні — LLM потребує людського судді на кожному кроці, вартість не знижується.

## Приклад

**`autoresearch` (Karpathy, березень 2026)**: ~630-рядків, AI-агенти самостійно ставлять ML-експерименти на одному GPU. Критерій верифікації — loss або швидкість навчання. За 2 дні знайшли 20 покращень, скоротили навчання GPT-2 на 11%.

## Як застосувати у Meweek

| Задача | Verifiable? | Обґрунтування |
|---|---|---|
| Генерація дайджесту | Частково (структура, секції) | Перевіряємо формат, але не якість змісту |
| weekly-review — класифікація задач | Так | Правило приналежності розділу перевіряється |
| Fitness — підбиття тижня | Так | Числові метрики (вага, повторення) верифікуються |
| Творче редагування нотаток | Ні | Суб'єктивна оцінка якості |
| DBManager — SQL-міграції | Так | Тести проходять або ні |

**Практичне правило**: перед створенням нової навички — записати критерій верифікації результату. Якщо критерій важко сформулювати, задача не готова до повної автоматизації.
