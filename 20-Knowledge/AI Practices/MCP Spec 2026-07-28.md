# MCP Spec 2026-07-28 — Stateless, Apps, Tasks

## Що це

Найбільше оновлення Model Context Protocol з моменту виходу. Release Candidate опублікований, фінальна специфікація — **28 липня 2026**.

## Ключові зміни

### Stateless Core
Сервери більше не потребують sticky sessions або shared session store. Будь-який MCP-сервер може розгортатись за звичайним round-robin load balancer.

**До:** MCP сервер → sticky session → DPI на gateway → складний деплой  
**Після:** MCP сервер → звичайний round-robin LB → production-ready

### MCP Apps Extension
Server-rendered UI всередині MCP клієнтів (Claude, Cursor тощо). Дозволяє серверам відображати інтерфейс прямо у вікні чату.

### Tasks Extension
Long-running асинхронні задачі. Агент може запустити завдання та отримати результат пізніше без блокування контексту.

### Authorization (OAuth 2.0 / OIDC)
Стандартизована авторизація для агент-до-агент взаємодії. Ближче до реальних enterprise OAuth deployments.

### MCP Server Cards
Автоматичний discovery серверів без ручного конфігу в `settings.json`. Сервери публікують свої можливості через domain-owned catalogs.

## Чому важливо

- Production MCP без DevOps костилів (sticky sessions, DPI)
- MCP Apps = нові UI в Claude без окремих веб-аплікацій
- Server Cards = plug-and-play замість ручного конфігу
- Tasks = агенти можуть запускати фонову роботу без блокування
