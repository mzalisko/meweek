# 🗄️ DBManager — Архівний код для флешки

Ця папка містить чисту копію коду проєкту **DBManager** для перенесення на флешку. Усі важкі та тимчасові файли видалено для економії місця та безпеки.

## Вміст папки

1. **`plugin/`** — WordPress-плагін `DBManager` (Must-Use фолбек у `mu/`, основний код у `dbmanager/`, тести та утиліти).
2. **`crm/`** — центральна система управління даними `DBManager Core` (адмінка, Livewire-компоненти, черги, API).
3. **`bridge/`** — ізоляційний проксі-шар `DataBridge` для подієвої синхронізації та захисту адреси CRM.

## Що було вилучено (очищено)

Для забезпечення чистоти та безпеки при перенесенні на флешку було вилучено:
- Папки залежностей `vendor/` та `node_modules/` (потрібно встановити заново).
- Локальні секрети `.env` (залишено тільки приклади конфігурації `.env.example`).
- Кеш та тимчасові файли тестів PHPUnit (`.phpunit.result.cache`).
- Логи роботи (`storage/logs/`).

## Як запустити проєкт після копіювання

1. **Для плагіна (`plugin/`):**
   - Перейдіть у `plugin/` та запустіть встановлення залежностей:
     ```bash
     composer install
     ```

2. **Для CRM (`crm/`):**
   - Перейдіть у `crm/` та запустіть встановлення PHP та JS залежностей:
     ```bash
     composer install
     npm install
     ```
   - Скопіюйте файл налаштувань та заповніть його:
     ```bash
     cp .env.example .env
     ```
   - Згенеруйте ключ додатка:
     ```bash
     php artisan key:generate
     ```

3. **Для мосту (`bridge/`):**
   - Перейдіть у `bridge/` та запустіть встановлення залежностей:
     ```bash
     composer install
     ```
   - Скопіюйте файл налаштувань та заповніть його:
     ```bash
     cp .env.example .env
     ```
   - Згенеруйте ключ додатка:
     ```bash
     php artisan key:generate
     ```
