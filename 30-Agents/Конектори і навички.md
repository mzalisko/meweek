# Конектори і навички

Що підключено до Claude (Cowork) у цьому середовищі та як цим користуватися. Службові гайди для агентів — у [[Гайди для агентів]].

## Підключені конектори (станом на 2026-06-11)

| Конектор | Можливості |
|---|---|
| GitHub (плагін engineering) | Робота з репозиторіями, PR, issues |
| Gmail | Пошук листів, чернетки, мітки |
| Notion | Сторінки, бази даних |
| Figma | Читання/створення дизайнів, діаграми FigJam |
| Google Drive | Пошук і читання файлів |
| Slack, Linear, Jira/Confluence, Datadog, PagerDuty, Asana | Доступні через плагін engineering (потребують автентифікації) |
| Claude in Chrome | Керування браузером |
| Computer use | Керування десктопними застосунками |

## Власні навички середовища (`.claude/skills/`)

| Навичка | Тригер | Що робить |
|---|---|---|
| `new-project` | «створи проєкт <Назва>» | Папка в `10-Projects/` з Огляд/Вимоги/План/Контекст + реєстрація в індексах |
| `save-context` | «збережи контекст» (перед /clear) | Оновлює `<Назва> — Контекст.md`, чекбокси плану, ADR, коміт |
| `weekly-review` | «розбери вхідні», «огляд тижня» | Розбір 00-Inbox, перевірка контекстів і посилань, архівація |
| `git-sync` | «синхронізуй», «запуш», старт/кінець сесії | pull --rebase перед роботою, commit+push після; конфлікти — з користувачем |
| `docker-project` | «зроби docker», «підніми оточення» | compose під проєкт: alpine, healthcheck, ліміти; секрети в .env поза git |
| `model-budget` | `/model-budget <задача>` | Підказує модель Claude під задачу; правила живуть у `.claude/skills/model-budget/SKILL.md` |
| `skill-forge` | `/skill-forge впровадь <ідея>` / `покращ <скіл>` | Перетворює ідеї на навички й оновлює людський гайд |
| `skill-creator` | «створи навичку», «покращ навичку» | Офіційна навичка Anthropic (вендорена в репо): створення, eval, оптимізація description |

Команди-кнопки до них — у `.claude/commands/` (`/new-project`, `/save-context`, `/weekly-review`, `/sync`, `/model-budget`, `/skill-forge`), людський опис — у [[Гайд — Команди Meweek]].

Формат — за практиками Anthropic ([[20-Knowledge/AI OS — практики Four Cs|деталі]]): коротка SKILL.md, «пушна» description з тригерами, правило + чому.

## Правило документації

Нова або змінена навичка не завершена, доки не оновлено пов'язані людські й службові записи:

1. запис у [[Гайд — Навички Meweek]];
2. запис у [[Гайд — Команди Meweek]], якщо є команда;
3. оновлення цього реєстру, якщо змінився перелік власних навичок;
4. коміт із усіма пов'язаними файлами.
