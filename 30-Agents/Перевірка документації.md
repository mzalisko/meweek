# Перевірка документації

Definition of Done для агентів перед комітом. Мета — щоб користувач не мусив вручну знаходити пропущені гайди, індекси й шаблони.

## Коли обов'язково запускати

- Додано або змінено команду, навичку, MCP, рутину чи workflow.
- Додано новий тип файлу в проєкті або профілі.
- Додано новий проєктний режим, формат звіту або повторювану структуру.
- Зміна має значення для того, як користувач щось запускає або де шукає результат.

## Перевірка

1. Людський гайд: чи є коротке пояснення для користувача в `50-Guides/` або проєктному `— Як працювати.md`.
2. Агентний контракт: чи є робоче правило для ШІ в `30-Agents/`, `AGENTS.md` або проєктному `— Команди.md`.
3. Індекс/огляд: чи додано посилання в центральний список (`Гайди.md`, `Проєкти.md`, проєктний `— Огляд.md`, реєстр профілів).
4. Шаблон/реєстр: якщо файл має повторюватися для нових сутностей, чи оновлено шаблон і обов'язковий набір.
5. Пошук: виконати `rg -n "<назва>|<команда>|<тип файлу>"` по релевантній папці й перевірити, що всі місця узгоджені.

## Для навичок

Нова або змінена навичка вважається завершеною тільки якщо:

- є `SKILL.md` або інший канонічний контракт;
- є тонка команда, якщо користувач має запускати її як `/команда`;
- є запис у [[Гайд — Навички Meweek простими словами]];
- якщо додано нову команду — оновлено [[Гайд — Команди Meweek простими словами]];
- оновлено [[Конектори і навички]], якщо змінився перелік власних навичок.

## Для 50-Guides

Гайд має відповідати людині, а не агенту:

- коротко: що це, коли використовувати, як запустити, як перевірити;
- без внутрішніх промптів, довгих правил агента й реалізаційних деталей;
- якщо технічні деталі потрібні, дати посилання на `30-Agents` або файл проєкту.

## Для 30-Agents

У `30-Agents` має потрапляти все, що потрібно знати ШІ:

- правила роботи агентів;
- контракти команд і навичок, якщо вони не належать конкретному проєкту;
- чеклісти перевірки перед комітом;
- реєстри підключень, навичок і службових процесів;
- ідеї та провенанс навичок.

Якщо новий текст потрібен агенту для правильної поведінки, але не потрібен користувачу як коротка інструкція, він не має жити в `50-Guides`.
