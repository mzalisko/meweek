# Ведення Obsidian

Це робочі правила для агентів, які редагують сховище Meweek як Obsidian vault.

## Перед редагуванням

1. Виконати `git status` і `git log --oneline -5`.
2. Якщо є незакомічені зміни не з поточної задачі — не чіпати їх.
3. Для проєкту читати спочатку `<Проєкт> — Контекст.md`, потім `<Проєкт> — Огляд.md`.
4. Не читати все сховище без потреби.

## Як створювати нотатки

1. Вибрати правильний розділ:
   - `00-Inbox` — чернетки й неструктуровані думки;
   - `10-Projects/<Назва>/` — проєктні матеріали;
   - `20-Knowledge` — сталі знання;
   - `30-Agents` — службові знання, правила, контракти й чеклісти для ШІ-агентів;
   - `50-Guides` — короткі гайди для користувача.
2. Називати `.md` файли українською.
3. Додавати frontmatter з тегами, якщо в розділі вже є така практика.
4. Додавати `[[wiki-посилання]]` на центральну нотатку.

## Як оновлювати проєкти

- `<Проєкт> — Контекст.md` тримати до 30 рядків.
- В огляді проєкту тримати список ключових файлів і журнал важливих змін.
- Великі деталі не складати в контекст: переносити в окремі нотатки.
- Якщо створено новий проєктний файл, додати його в `<Проєкт> — Огляд.md`.
- Якщо створено новий тип файлу або повторюваний workflow, оновити відповідний шаблон, реєстр або контракт команд.
- Якщо зміна важлива для користувача або користувач просить “опиши/поясни/зроби гайд”, оновити відповідний людський гайд у `50-Guides/`.

## Розділення аудиторій

- `30-Agents` — усе, що потрібно знати ШІ: правила, контракти, чеклісти, внутрішні інструкції, реєстри навичок.
- `50-Guides` — тільки короткі гайди для користувача: що це, коли запускати, як перевірити.
- Якщо текст звертається до агента словами “агент має”, “перед комітом перевір”, “обов'язково онови” — це не гайд, а файл для `30-Agents` або проєктний контракт.
- Проєктні контракти — у `10-Projects/<Назва>/`, наприклад `Fitness — Команди.md`.

## Граф Obsidian

- Граф формується автоматично з посилань.
- Не редагувати `.obsidian/` для косметики графа без явного прохання.
- Щоб покращити граф, додавати змістовні зв'язки між нотатками.
- Сирітські нотатки або прив'язувати до індексу/огляду, або залишати в `00-Inbox` до розбору.

## Коміти

- Комітити тільки файли поточної задачі.
- Коміт українською: `розділ: що зроблено`.
- Не пушити без прямого прохання.
