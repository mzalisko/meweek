# Пропозиція навички: billing-monitor

## Назва навички
`billing-monitor`

## Мета
Перевіряти поточний стан використання кредитів Anthropic та попереджати про наближення до ліміту. Особливо критично після 15 червня 2026, коли Agent SDK, `claude -p` та GitHub Actions перейдуть на окремі кредити за API-тарифами.

## Вхідні дані
- Опціонально: поріг попередження у % (за замовчуванням 80%)
- Опціонально: тип плану (Pro / Max 5x / Max 20x)

## Вихідні дані
- Поточний стан кредитів (використано / доступно)
- Попередження якщо > порогу
- Список автоматизацій, що витрачають агентні кредити
- Рекомендації по оптимізації

## Приклад воркфлоу

```
/billing-monitor
→ Перевіряє platform.anthropic.com/settings/billing
→ Виводить: "Використано 45% кредитів. Залишилось $11.00 з $20.00"
→ Якщо > 80%: "⚠️ Наближаєтесь до ліміту. Топ витрат: daily digest (40%), weekly review (25%)"
```

## Оцінка цінності
- **Висока**: запобігає несподіваним витратам після зміни billing 15 червня
- Одноразове налаштування з регулярним використанням

## Складність реалізації
- **Низька-Середня**: потребує доступу до Anthropic API або web UI для отримання даних білінгу
- Обмеження: Anthropic не має публічного API для billing stats — може потребувати WebFetch до dashboard або ручного введення даних

## Альтернатива
Більш проста версія: навичка, що виводить список всіх автоматизацій (`claude -p` скриптів, GitHub Actions) та оцінює їх орієнтовну вартість на основі частоти запуску.
