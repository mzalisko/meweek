# Skill Proposal: idea-file-share

## Назва навички
`idea-file-share`

## Мета
Перетворити будь-яку задачу або концепцію на **Idea File** — стандартизований, абстрактний опис, який будь-який LLM-агент може отримати і реалізувати під свій контекст. Патерн запропонований Андрієм Карпаті (червень 2026).

## Вхідні дані (Inputs)
- `topic` — тема або задача для опису (наприклад, «навичка для обробки Inbox», «workflow для щотижневого огляду»)
- `audience` — для кого (агент з Obsidian, агент без контексту, загальний)

## Вихідні дані (Outputs)
- Idea File у форматі `.md` у `30-Agents/Skill Ideas/` або в окремій директорії
- Файл: абстрактний і дієвий опис без жорсткої прив'язки до інструментів, достатній для агента будь-якої платформи

## Приклад воркфлоу

```
Користувач: /idea-file-share topic="weekly review для Obsidian сховища"

Навичка:
1. Аналізує контекст задачі
2. Формує абстрактний Idea File (мета, вхід, вихід, приклад, варіації)
3. Зберігає в 30-Agents/Skill Ideas/
4. Не прив'язується до Claude/Obsidian — агент-одержувач адаптує сам
```

## Структура Idea File

```markdown
# Ідея: [назва]

## Мета
[одне речення про те, що вирішує ця ідея]

## Вхід
[що потрібно мати/надати агенту]

## Вихід
[що агент видає]

## Приклад сценарію
[абстрактний сценарій без прив'язки до конкретних інструментів]

## Варіації
[як ідею можна адаптувати під різні контексти]
```

## Оцінка цінності
- Спрощує обмін практиками між агентами (Claude ↔ Codex ↔ Gemini)
- Навички Meweek можна описувати як Idea Files і легко переносити між платформами
- Скорочує час на «переклад» досвіду між сесіями

## Складність реалізації
**Low** — по суті, шаблонний промпт + стандартизована структура файлу
