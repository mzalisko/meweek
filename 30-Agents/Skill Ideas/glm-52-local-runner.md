# Ідея навички: glm-52-local-runner

## Назва навички
`glm-52-local-runner`

## Мета

Запустити GLM-5.2 локально (через Ollama або HuggingFace Transformers) для конфіденційних задач, де небажано передавати дані на зовнішні API (в т.ч. китайські сервери Z.ai).

## Тригер

Коли користувач каже: «запусти локально», «конфіденційна задача», «не хочу API», «GLM-5.2 офлайн».

## Вхідні дані

- Задача або промпт
- Опціонально: максимальна кількість токенів відповіді
- Опціонально: прапорець `--no-think` для швидкого режиму

## Вихідні дані

- Відповідь моделі
- Інформація про час виконання і використану пам'ять

## Приклад воркфлоу

1. Перевірити чи є Ollama / `glm-5.2` модель встановлена
2. Якщо немає — запропонувати команду для pull
3. Запустити запит з вхідним промптом
4. Повернути результат у чат

## Оцінена цінність

Середня. Корисна для проєктів з чутливими даними (DBManager, особисті нотатки) де не хочеться витрачати API-кредити на стандартні задачі типу резюмування або класифікації.

## Складність реалізації

Low-Medium. Потребує Ollama або Python-оточення. GLM-5.2 у повному розмірі (753B) потребує ~400GB VRAM; є quant-версії (Q4_K_M ~220GB). Для більшості домашніх машин реалістично лише через хмарний GPU або сильно квантизовану версію.

## Примітка щодо ризику

GLM-5.2 API (cloud) несе ризик даних через китайське законодавство. Локальний запуск знімає цей ризик. MIT-ліцензія дозволяє будь-яке комерційне використання.
