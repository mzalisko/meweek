# Skill Idea: loop-verifier

## Назва навички
`loop-verifier`

## Мета
Перетворити будь-яку разову задачу на надійний агентний цикл з автоматичною верифікацією результату. Агент працює ітеративно до досягнення критерію якості — без ручного втручання між тиками.

## Інпути
- `task`: опис задачі (рядок або YAML-блок)
- `criterion`: критерій завершення — тест, grep, перевірка файлу, запит до API
- `max_iterations`: максимум спроб (default: 5)
- `model`: модель для виконання (default: claude-sonnet-4-6)

## Аутпути
- Статус: `done` / `failed` / `max_iterations_reached`
- Кількість ітерацій
- Фінальний результат або опис останньої помилки

## Приклад воркфлоу

```
Вхід: task="рефактор функції parse_csv", criterion="всі тести зелені (pytest tests/)", max_iterations=5

Тик 1: агент рефакторить → pytest → 2 тести падають → criterion не виконано
Тик 2: агент виправляє → pytest → всі зелені → criterion виконано → DONE (2 ітерації)
```

## Оцінка цінності
- **Висока**: знімає потребу стежити за агентом між кроками; дозволяє запускати задачі "фоново" і повертатись до результату.
- Особливо корисно для: code refactor, тест-фікси, генерація контенту з вимогами до якості.

## Складність впровадження
**Medium** — потрібен shell-runner для виконання criterion-команди + передача stdout назад агенту як context.

## Зв'язок із наявними навичками
- Схожа логіка є в `weekly-review` (перевіряє умови завершення) — можна витягти спільний pattern.
- Базується на концепції з [[Loop Engineering — дизайн циклів]].
