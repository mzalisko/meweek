# Ідея навички: quiet-mode

## Назва скілу

`quiet-mode`

## Мета

Автоматично вмикати "тихий режим" на початку автоматизованої сесії та вимикати після завершення. Агент не надсилає системних push-сповіщень — лише цілеспрямовані через PushNotification tool.

## Тригери

Використовувати, коли:
- Рутина запускається автоматично (за розкладом)
- Сесія є фоновою (без живого користувача)
- Потрібно уникнути зайвого шуму від Claude Code системних сповіщень

## Inputs

- Шлях до файлу-маркера (опціонально, за замовчуванням `/tmp/claude-presence-{PID}`)
- Дія: `start` | `stop`

## Outputs

- Файл-маркер створено/видалено
- `CLAUDE_CLIENT_PRESENCE_FILE` встановлено в середовищі

## Приклад workflow

```
SessionStart хук → quiet-mode start
... рутина виконується ...
→ PushNotification якщо є важливе
SessionStop хук → quiet-mode stop (прибирає файл)
```

## Оціночна цінність

- **Висока**: усуває системний шум для всіх рутин Meweek одразу
- Підвищує якість сповіщень — коли приходить push, він завжди важливий

## Складність реалізації

- **Низька**: один bash-виклик на start/stop
- Інтегрується через SessionStart/SessionStop хуки в `settings.json`
