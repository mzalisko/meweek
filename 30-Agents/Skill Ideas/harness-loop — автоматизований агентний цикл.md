# Skill Proposal: harness-loop

## Назва навички
`harness-loop`

## Мета
Запускати будь-яку навичку або команду Claude Code у режимі headless (`claude -p`) як частину cron-керованого агентного циклу. Реалізує патерн Harness Engineering від Борисa Черні.

## Вхідні дані (Inputs)
- `command` — навичка або промпт для виконання (наприклад, `/daily-digest`, `/weekly-review`)
- `schedule` — cron-вираз (наприклад, `0 7 * * *`)
- `working_dir` — директорія запуску (за замовчуванням: `/home/user/meweek`)
- `log_file` — шлях до лог-файлу
- `max_tokens` — ліміт токенів на виклик (захист від перевитрат)

## Вихідні дані (Outputs)
- Виконана дія відповідно до переданої команди
- Запис результату у лог-файл
- Оновлені файли контексту/нотаток у сховищі

## Приклад воркфлоу

```
Користувач: /harness-loop command=/daily-digest schedule="0 7 * * *"

Навичка:
1. Перевіряє наявний cron у системі
2. Додає запис: 0 7 * * * claude -p "/daily-digest" >> ~/meweek/logs/digest.log
3. Створює logs/ директорію якщо відсутня
4. Підтверджує активацію
```

## Оцінка цінності
- **Автоматизація**: щоденний дайджест, weekly review, Fitness звіти — без ручного запуску
- **Масштаб**: один harness може координувати кілька навичок послідовно
- **Економія часу**: ~15–30 хвилин/тиждень на ручних запусках рутин
