# Ідея навички: changelog-monitor

## Назва навички
`changelog-monitor`

## Мета
Автоматично відстежувати оновлення Claude Code (та інших ключових інструментів) і зберігати нові фічі у базі знань Meweek без ручного моніторингу.

## Вхідні дані
- Список URL для моніторингу (changelog, release notes, GitHub releases)
- Дата останньої перевірки (зберігається у файлі стану)
- Директорія для збереження знахідок (`20-Knowledge/AI Practices/`)

## Вихідні дані
- Оновлений файл знань з новими фічами
- Повідомлення про зміни (якщо є)
- Оновлена дата останньої перевірки

## Приклад workflow

```
1. Прочитати дату останньої перевірки з .claude/changelog-state.json
2. Завантажити changelog (code.claude.com/docs/en/changelog)
3. Знайти нові записи після дати останньої перевірки
4. Якщо є нові:
   a. Класифікувати по категоріях (ffix / feature / breaking)
   b. Оновити відповідну нотатку в 20-Knowledge/AI Practices/
   c. Сформувати короткий список нових фіч
5. Оновити дату у .claude/changelog-state.json
6. Повернути список нових фіч або "без змін"
```

## Орієнтовна цінність
- **Висока**: замість ручного перегляду changelog раз на тиждень — автоматично фіксує кожну нову фічу в день появи.
- Інтегрується з `daily-digest` рутиною: замість пошуку по веб, моніторить першоджерело.
- Можна розширити на: OpenAI release notes, MCP spec, n8n changelog.

## Складність реалізації
**Низька** — WebFetch + порівняння дат + Write до файлу. Без складних залежностей.
