# Skill Idea: subagent-cost-audit

## Назва навички
`subagent-cost-audit`

## Мета
Аналізує поточні субагентні воркфлоу в навичках Meweek і рекомендує оптимальну модель Claude для кожного рівня ієрархії (Opus/Sonnet/Haiku) на основі складності та типу задачі. Після billing split (червень 2026) це прямий вплив на витрати.

## Вхідні дані
- Список навичок або команди для аудиту (або `all` для всіх)
- Поточна конфігурація моделей (з `.claude/settings.json`)

## Вихідні дані
- Таблиця: навичка → рівень → поточна модель → рекомендована модель → орієнтовна економія
- Готовий блок для `settings.json` або shell export
- Список навичок, де зміна безпечна без ризику якості

## Приклад воркфлоу
```
/subagent-cost-audit all

→ Аналіз weekly-review: root=Opus ✓, рівень 1=Opus ⚠️ (рекомендовано Sonnet), leaf=Opus 🔴 (рекомендовано Haiku)
→ Оцінена економія: ~60% на leaf-задачах
→ Пропозиція: export CLAUDE_CODE_SUBAGENT_MODEL=claude-haiku-4-5-20251001
```

## Орієнтована цінність
- **Висока**: після billing split кожен субагент-виклик коштує за API-ставкою. Leaf-задачі (читання файлів, grep) на Opus — 10-20x переплата.
- Окупається вже після першого weekly-review або daily-digest.

## Складність реалізації
**Низька**: навичка читає SKILL.md файли навичок, визначає рівні субагентів, зіставляє з типовими паттернами (читання = Haiku, синтез = Opus). Без зовнішніх API, без змін у файлах без підтвердження.
