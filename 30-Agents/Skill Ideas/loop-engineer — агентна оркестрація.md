# Ідея навички: loop-engineer

## Назва
`loop-engineer`

## Призначення
Перетворює одну задачу на багаторівневий агентний цикл: координатор сам пише план оркестрації, визначає субагентів і запускає їх паралельно. Реалізує концепцію Loop Engineering (Boris Cherny, червень 2026).

## Вхідні дані
- Опис задачі (наприклад: "зроби weekly-review", "проаналізуй Fitness тиждень")
- Опціонально: модель для субагентів, максимальна глибина (1-5)

## Вихідні дані
- Готовий план оркестрації (YAML або markdown)
- Автоматичний запуск субагентів
- Агрегований результат

## Приклад workflow

```
/loop-engineer зроби weekly-review

→ Координатор аналізує сховище (git log, структура проєктів)
→ Пише план:
    - Субагент-1: Fitness зміни цього тижня
    - Субагент-2: DBManager зміни
    - Субагент-3: нові Skill Ideas
    - Субагент-4: вхідні в 00-Inbox
→ Запускає паралельно (Dynamic Workflows)
→ Збирає підсумки → weekly-review.md
```

## Орієнтовна цінність
- Висока: скорочує час weekly-review з 30+ хв до 5-10 хв.
- Якість вища за одиничний агент: кожен субагент фокусований.
- Готується до роботи з Claude Code Dynamic Workflows (Enterprise).

## Складність впровадження
Середня. Потрібно:
1. Написати SKILL.md з шаблоном координаційного промпту.
2. Визначити mapping задача → субагенти для типових задач Meweek.
3. Протестувати на weekly-review.
