# Skill Proposal: dynamic-workflow-harness

## Назва навички
`dynamic-workflow-harness`

## Мета
Перетворити існуючі Meweek-рутини (`/daily-digest`, `/weekly-review`, `/fitness`) на Dynamic Workflows із паралельними субагентами через `ultracode` keyword — скорочення часу виконання в 3–5 разів.

## Вхідні дані (Inputs)
- `skill` — назва рутини або команди (наприклад, `daily-digest`, `weekly-review`)
- `parallel_tasks` — список незалежних підзадач для паралелізації
- `model` — опційно: `claude-fable-5` для складних задач, `claude-haiku-4-5` для простих
- `max_depth` — максимальна глибина субагентів (1–5, за замовчуванням 2)

## Вихідні дані (Outputs)
- Синтезований результат рутини з паралельних потоків
- Метадані: кількість субагентів, час виконання, використані моделі
- Збережені файли відповідно до оригінальної рутини

## Приклад воркфлоу

```
Користувач: /dynamic-workflow-harness skill=daily-digest

Навичка:
1. Аналізує рутину daily-digest → виділяє незалежні пошукові задачі
2. Будує ultracode-промпт з явними паралельними гілками:
   - Гілка A: пошук по Anthropic/Claude/Boris Cherny
   - Гілка B: пошук по OpenAI/Andrej Karpathy
   - Гілка C: пошук по MCP/GitHub AI
   - Гілка D: пошук по Simon Willison/Nate Herk
3. Запускає мульти-агентний ранінг через ultracode
4. Синтезує результати у стандартному форматі дайджесту
5. Зберігає у 30-Agents/AI News Digest/Daily/
```

## Порівняння з поточним підходом

| | Поточний (sequential) | Dynamic Workflow |
|---|---|---|
| Час пошуку | ~15 хв (4–6 серч по черзі) | ~4 хв (паралельно) |
| Покриття джерел | Залежить від контексту | Гарантоване по всіх гілках |
| Якість синтезу | Одна модель | Оркестратор + спеціалізовані агенти |

## Оцінка цінності
- **Економія часу**: ~10 хв/день (3650 хв/рік = ~60 годин)
- **Якість**: вищий coverage, менше пропущених джерел
- **Масштабованість**: легко додати нові гілки (Obsidian, productivity, тощо)

## Складність реалізації
**Medium** — потребує:
- Розуміння Dynamic Workflows API
- Рефакторинг існуючих рутин під паралельну структуру
- Тестування на реальних сесіях (можливі витрати кредитів)

## Застереження
- Dynamic Workflows — GA, але потребує `ultracode` keyword
- Вартість кредитів зросте (більше субагентів)
- Рекомендовано спочатку протестувати на `weekly-review` (нижча частота = менші витрати)
