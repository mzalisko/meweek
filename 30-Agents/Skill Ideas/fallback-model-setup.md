# Ідея навички: fallback-model-setup

## Призначення

Налаштування ланцюга резервних моделей (`fallbackModel`) у `.claude/settings.json` для підвищення надійності автоматичних рутин і агентних сесій.

## Вхідні дані

- Основна модель (default: claude-opus-4-8)
- Список резервних моделей у порядку пріоритету

## Вихідні дані

- Оновлений `.claude/settings.json` з `fallbackModel` конфігурацією
- Підтвердження налаштування

## Приклад workflow

```
/fallback-model-setup
→ Перевіряє поточні settings.json
→ Пропонує оптимальний ланцюг: opus-4-8 → sonnet-4-6 → haiku-4-5
→ Записує конфігурацію
→ Тестує доступність кожної моделі
```

## Розширений сценарій

Навичка може аналізувати поточні рутини і пропонувати різні ланцюги для різних задач:
- Важкі задачі: opus → sonnet
- Швидкі задачі: sonnet → haiku
- Рутини: sonnet → haiku (cost-efficient)

## Орієнтовна цінність

- **Висока**: рутини не падають при перевищенні лімітів основної моделі
- Особливо важливо для нічних і ранкових рутин (цей дайджест)
- 5 хвилин налаштування = захист всіх автоматичних процесів

## Складність реалізації

**Low** — читання/запис JSON, без зовнішніх залежностей
