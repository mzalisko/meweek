# Ідея навички: dynamic-workflow-builder

## Назва

`dynamic-workflow-builder`

## Мета

Допомогти проектувати Dynamic Workflows для Claude Code: приймає опис великої задачі і повертає готовий план паралельної оркестрації — список підзадач, розподіл агентів, умови верифікації, стратегію синтезу.

## Вхід

- Опис задачі (рядком): що потрібно зробити
- Контекст (опційно): які файли/папки зачіпаються, які обмеження
- Бажаний патерн (опційно): split-and-merge / agent-teams / headless

## Вихід

Структурований план Dynamic Workflow у форматі:
1. **Фаза fan-out**: список паралельних підзадач + модель для кожної (Haiku/Sonnet/Opus)
2. **Фаза reduce**: правила агрегації результатів
3. **Фаза synthesize**: що робить оркестратор з результатами
4. **Умова завершення**: як перевіряємо успіх
5. **Оцінка токенів**: приблизно скільки витратимо

## Приклад воркфлоу

```
/dynamic-workflow-builder "Переглянь всі проєкти Meweek і оновлення Контекст.md кожного"

→ Fan-out: 5 агентів (Haiku) — по одному на кожен проєкт
→ Reduce: список змін по кожному проєкту
→ Synthesize: Sonnet оновлює Проєкти.md з загальним статусом
→ Верифікація: кожен Контекст.md оновлено за останні 24 год
→ Оцінка: ~50K токенів
```

## Оцінка цінності

- **Висока**: Dynamic Workflows — нова можливість Claude Code, але без структурованого підходу до проектування легко витратити зайві токени або отримати неякісний результат
- Особливо цінно для Meweek, де є багато схожих повторюваних задач (огляди проєктів, fitness-звіти, inbox processing)

## Складність реалізації

**Низька**: навичка є шаблоном промпту + базовою логікою класифікації задачі. Не потребує зовнішніх інструментів.
