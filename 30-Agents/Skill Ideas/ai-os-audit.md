# Ідея навички: ai-os-audit

## Назва
`ai-os-audit`

## Мета
Щотижневий аудит AI Operating System Meweek: виявляє прогалини в автоматизації, деградуючі навички та нові можливості автоматизації, які з'явились за тиждень.

## Натхнення
Концепція `/audit` з Nate Herk's AIS-OS (github.com/nateherkai/AIS-OS) — gap-звіт архітектури системи, адаптований для Meweek.

## Вхідні дані (Inputs)
- Git log за останній тиждень (`git log --oneline --since="7 days ago"`)
- Список наявних навичок (`.claude/skills/`)
- Список ідей навичок (`30-Agents/Skill Ideas/`)
- Список рутин (`.claude/settings.json` CronList або `30-Agents/`)
- Опційно: список останніх завдань (TaskList)

## Вихідні дані (Outputs)
Звіт у `30-Agents/AI OS Audit/YYYY-MM-DD.md` з секціями:

1. **Задачі, що повторювались двічі і більше** — кандидати на нову навичку
2. **Навички, що не використовувались >2 тижні** — можливо застаріли або не знаходяться
3. **Нові ідеї з практики** — що робилось вручну, що можна автоматизувати
4. **Gap в автоматизації** — порівняння todo-list і наявних навичок
5. **Рекомендований наступний крок** — одна конкретна дія

## Приклад воркфлоу

```
/ai-os-audit
→ читає git log, список skills, Skill Ideas, рутини
→ порівнює: "що робилось вручну за тиждень" vs "що вже автоматизовано"
→ генерує gap-звіт
→ пропонує 1-3 конкретні наступні кроки (нова навичка / оновлення рутини / видалення застарілого)
```

## Орієнтовна вартість впровадження
- **Складність**: Low (читання файлів + аналіз + запис звіту)
- **Час**: 2–4 год на реалізацію
- **Запуск**: щотижня вручну або автоматично (CronCreate, наприклад неділя 08:00)

## Оцінка цінності
- **Impact**: Medium-High — виявляє "сліпі плями" в автоматизації
- **Effort**: Low — аналіз вже існуючих даних
- **Унікальність**: немає аналогу в наявних навичках Meweek

## Критерій успіху (Verifiable Output)
Звіт містить >= 1 конкретну рекомендацію → перевіряється структурно. Якість рекомендацій — перевіряється користувачем при щотижневому огляді.

## Зв'язки
- Доповнює `weekly-review` — ширший огляд сховища vs глибший аудит AI-системи
- Використовує результати `save-context` для розуміння що робилось
- Результати стають входом для `skill-forge`
