# Вибір моделей

Службова матриця для агентів Meweek. Використовувати, коли треба порадити модель Claude під задачу або налаштувати рутину.

## Матриця

| Тип задачі | Модель | Чому |
|---|---|---|
| Інтерактивна розробка, складна архітектура, критичний код DBManager | `claude-fable-5` або `claude-opus-4-8` | Максимальна якість; ціна виправдана там, де помилка дорога |
| Автоматичні рутини: дайджест, тижневий огляд | `claude-sonnet-4-6` | Якості достатньо, дешевше за Opus/Fable |
| Шаблонна генерація: Fitness-звіти, стандартні підсумки | `claude-haiku-4-5` | Структурований вивід, мінімум міркувань, найшвидше й найдешевше |
| Масова проста обробка, класифікація | `claude-haiku-4-5` | Найдешевше за токен |
| Code review / security review | `claude-sonnet-4-6`; для DBManager-безпеки — `claude-opus-4-8` або `claude-fable-5` | Баланс; критичне варте дорожчої моделі |

## Відносна вартість

Орієнтир станом на 2026-06, API-прайс за вихідні токени:

| Модель | $/1M output | Відносно Haiku |
|---|---|---|
| `claude-haiku-4-5` | $5 | 1× |
| `claude-sonnet-4-6` | $15 | 3× |
| `claude-opus-4-8` | $25 | 5× |
| `claude-fable-5` | $50 | 10× |

Перед фінансовими рішеннями перевірити актуальні ціни через офіційне джерело або навичку `claude-api`, бо лінійка й тарифи змінюються.

## Як відповідати

1. Визначити тип задачі.
2. Назвати рекомендовану модель і коротко пояснити чому.
3. Дати відносну вартість, якщо це доречно.
4. Для рутини підказати готовий рядок `"model": "claude-..."`.
5. Якщо даних недостатньо, поставити уточнення замість вигадування.
