## 🤖 Claude Code та агенти

**Реліз 2.1.172 (10 червня) — ключові зміни:**
- Субагенти → власні субагенти, до 5 рівнів рекурсії
- Bedrock: регіон тепер читається з `~/.aws` (не тільки `AWS_REGION`)
- Пошук у `/plugin` marketplace
- Фікс: сесії з 1M контекстом без кредитів більше не зависають — авто-компакт
- Фікс: `WebFetch(domain:*.example.com)` тепер коректно матчить subdomains

**Реліз 2.1.173 (11 червня) — мінорні фікси:**
- Нормалізація імен Fable 5 з суфіксом `[1m]`

**Реліз 2.1.169 (8 червня) — DevEx:**
- `/cd` — змінює робочу директорію без розриву prompt cache
- `--safe-mode` / `CLAUDE_CODE_SAFE_MODE` — запуск без всіх кастомізацій
- `disableBundledSkills` — контроль над вбудованими навичками

## Для Meweek

1. **Що варто спробувати сьогодні?** — Оновити модель в Claude Code до Fable 5 та протестувати рекурсивні субагенти в одній з навичок (наприклад, `weekly-review`).

2. **Яку нову навичку додати в `.claude/skills`?** — `billing-monitor`: скрипт для перевірки поточного використання кредитів та попередження перед досягненням ліміту.

3. **Яку рутину можна автоматизувати?** — `post-session` хук для автоматичного `git add + commit + push` після завершення кожної сесії Claude Code. Зараз це ручний крок.

4. **Який MCP сервер заслуговує уваги?** — `aaronsb/obsidian-mcp-plugin` — дозволяє агентам читати/писати vault з підтримкою wikilinks та Dataview. Прямий профіт для Meweek.

5. **Яка одна ідея сьогодні дасть найбільшу економію часу?** — `post-session` git хук. Одноразове налаштування в settings.json, а потім кожна сесія автоматично зберігає стан. Зараз `/sync` потрібно запускати вручну.
