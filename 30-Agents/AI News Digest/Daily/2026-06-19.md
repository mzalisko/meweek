## 🔴 Найважливіше

### 1. 🔴 Loop Engineering — новий стандарт роботи з агентами (Boris Cherny)
Борис Черні (голова Claude Code) офіційно закріпив поняття **Loop Engineering**: замість ручного промптинґу — пишеш цикл, який сам промптує агента, читає результат, вирішує, чи задача завершена, і якщо ні — промптує знову. Термін спільно сформулювали Черні та Адді Osmani (Google Chrome) ~16 червня. До грудня 2025 Черні видалив свій IDE; 100% його коду за останні 30 днів написав Claude Code.

- **Ключова цитата**: "My job is to write loops, not prompts."
- **Вплив**: змінює ментальну модель роботи — людина задає критерій якості, агент iterates до досягнення.
- 🟢 Ready to Apply | Impact: High | Effort: Medium | Confidence: High
- **Дія**: прочитати нотатку `20-Knowledge/AI Practices/Loop Engineering — дизайн циклів.md`

## Для Meweek

**1. Що варто спробувати сьогодні?**
Налаштувати `fallbackModel` у `.claude/settings.json` — 5 хвилин, захищає всі рутини від зупинки через rate limit або недоступність моделі.

**2. Яку нову навичку додати в `.claude/skills`?**
`loop-verifier` — навичка, що приймає задачу + критерій завершення → запускає агентний цикл → сама перевіряє результат → виводить "done / retry". Перетворює будь-яку одноразову задачу на надійний агентний процес.

**3. Яку рутину можна автоматизувати?**
Перевірку наявності нових MCP серверів під Obsidian — простий щотижневий agent loop: пошук по GitHub topics `obsidian-mcp` → порівняння з відомими → нотифікація якщо є нові.

**4. Який MCP сервер заслуговує уваги?**
`Obsilo Agent` (Obsidian) — 55+ інструментів з 3-tier memory дозволить агентам Meweek пам'ятати контекст між сесіями без ручного `— Контекст.md` файлу.

**5. Яка одна ідея сьогодні дасть найбільшу економію часу?**
Додати `/cd` у workflow рутин, де потрібна зміна директорії: замість `restart session → cd → rebuild cache` — одна команда зберігає кеш і економить ~30 сек на кожному переключенні.
