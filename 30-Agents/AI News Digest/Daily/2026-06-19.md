## 🔴 Найважливіше

1. **Claude Code: `/config key=value` прямо з промпту** — нова команда дозволяє змінювати будь-яке налаштування без редагування файлів: `/config thinking=false`. Працює в інтерактивному режимі, `-p` та Remote Control. Прямий вплив на всі наші рутини та скіли.

2. **`CLAUDE_CLIENT_PRESENCE_FILE` — керування мобільними сповіщеннями** — нова змінна середовища: якщо файл існує, Claude Code не надсилає push-сповіщення на телефон. Ідеально для автоматизованих рутин, щоб не спамити під час фонової роботи.

3. **Claude Code `--safe-mode`** — запуск без усіх кастомізацій (CLAUDE.md, плагіни, скіли, хуки, MCP). Критично важливо для відлагодження, коли щось ламається і не зрозуміло що.

4. **Karpathy: Software 3.0** — у червні 2026 Karpathy сформулював нову тезу: AI автоматизує все, що людина може верифікувати (тести, оцінка гри, формальний доказ). Практичний критерій для оцінки, чи варто автоматизувати задачу.

5. **MCP: статус протоколу** — 97M+ щомісячних завантажень SDK, 81K+ зірок GitHub, 500+ публічних серверів. MCP стає беззаперечним стандартом для AI-інтеграцій.

## Для Meweek

**1. Що варто спробувати сьогодні?**
Налаштувати `CLAUDE_CLIENT_PRESENCE_FILE` для автоматизованих рутин — виключити системний шум і перейти на керовані push через PushNotification tool.

**2. Яку нову навичку додати в `.claude/skills`?**
`quiet-mode` — скіл, що встановлює "presence file" на початку сесії та прибирає його при завершенні. Автоматично тихий режим для будь-якої рутини.

**3. Яку рутину можна автоматизувати?**
Автоматична перевірка MCP-серверів на сумісність після виходу нових версій протоколу — особливо актуально до RC (2026-07-28).

**4. Який MCP сервер заслуговує уваги?**
Obsidian CLI як MCP-сервер — дає агентам повноцінний доступ до vault через wikilink-граф та Dataview замість прямих file reads.

**5. Яка одна ідея сьогодні дасть найбільшу економію часу?**
`CLAUDE_CLIENT_PRESENCE_FILE` + PushNotification tool — чіткий розподіл між "фоновий шум" і "справжнє сповіщення". Менше переривань, більше сигналу.
