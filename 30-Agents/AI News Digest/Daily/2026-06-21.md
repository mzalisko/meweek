## 🔴 Найважливіше

### 1. 🔴 Dynamic Workflows у Claude Code — офіційний GA

Anthropic оголосив про загальний доступ до **Dynamic Workflows** — нова можливість Claude Code для складних задач через координацію великої кількості AI-агентів у межах одного воркфлоу.

**Як працює:** Claude динамічно генерує скрипт оркестрації → розбиває задачу на підзадачі → запускає агентів паралельно → верифікує результати → ітерує до збіжності.

**Реальний результат:** Jarred Sumner за допомогою Dynamic Workflows портував Bun із Zig на Rust — **750 000 рядків Rust**, 99.8% тестового набору пройшло, **11 днів від першого коміту до merge**.

**5 базових патернів:** sequential, operator, split-and-merge, agent teams, headless. Узагальнений шаблон: `fan out → reduce → synthesize`.

- **Увага**: Dynamic Workflows витрачають суттєво більше токенів. Починати з обмеженої задачі.
- 🟡 Review First | Impact: High | Effort: Medium | Confidence: High

### 2. 🔴 Claude Code Auto Mode: блокування деструктивних git-команд

Нове захисне правило в auto-режимі: Claude Code тепер блокує:
- `git reset --hard`, `git checkout -- .`, `git clean -fd`, `git stash drop` — якщо ви не просили прибирати локальну роботу
- `git commit --amend` — якщо коміт не зробив агент у поточній сесії
- `terraform/pulumi/cdk destroy` — якщо не запитано конкретний стек

Це прямо зачіпає Meweek: агенти в headless-рутинах (`daily-digest`, `weekly-review`) тепер безпечніші.

- 🟢 Ready to Apply | Impact: High | Effort: Low | Confidence: High

## 🤖 Claude Code та агенти

**Головний патерн тижня — Dynamic Workflows:**

```
# Концептуальний harness для великої міграції
claude --workflow "Портуй всі Python 2 скрипти на Python 3 у /src" \
  --parallel-agents 20 \
  --verify "pytest tests/"
```

**Boris Cherny's loop philosophy:**
- Не промптити Claude напряму
- Писати loops, що запускають агентів автоматично
- Верифікувати результат, а не детально інструктувати процес

**Karpathy + Cherny = одна ідея:** якщо можеш перевірити результат — можеш делегувати задачу AI.

### Dynamic Workflows: fan-out → reduce → synthesize

- **Пріоритет**: 🔴 Critical (для великих задач)
- **Impact**: High | **Effort**: Medium | **Confidence**: High
- **Статус**: 🟡 Review First
- **Чому важливо**: Single-thread Claude Code не масштабується на задачі типу «переглянь 50 файлів і знайди проблеми». Dynamic Workflows запускає 20+ агентів паралельно.
- **Де fits**: weekly-review (паралельний огляд всіх проєктів), code-review великих PR, міграції

- **Пріоритет**: 🟡 Useful
- **Impact**: High | **Effort**: Low | **Confidence**: High

## Для Meweek

**1. Що варто спробувати сьогодні?**
Dynamic Workflows для `weekly-review` — замість лінійного перегляду запустити паралельний: кожен проєкт окремим агентом, потім синтез. Це може скоротити час огляду з 15 до 3–4 хвилин.

**2. Яку нову навичку додати в `.claude/skills`?**
`dynamic-workflow-builder` — навичка, що на вхід отримує опис великої задачі і генерує готовий Dynamic Workflow план (список паралельних підзадач, умови верифікації, стратегію синтезу).

**3. Яку рутину можна автоматизувати?**
Auto-перевірка статусу billing split — раз на день перевіряти чи headless режим досі на підписці або перейшов на кредити. Поки не перейшов — нічого робити не треба; коли перейде — повідомити.

**4. Який MCP сервер заслуговує уваги?**
`mcpvault` (bitbonsai) — BM25 пошук по Obsidian vault без будь-яких залежностей і без потреби тримати Obsidian запущеним. Ідеально для headless рутин у хмарному середовищі.

**5. Яка одна ідея сьогодні дасть найбільшу економію часу?**
Застосувати принцип SW 3.0 до наявних задач: замість детальних інструкцій — чітка умова верифікації + делегування. Наприклад, для Fitness-звітів: «Підбий тиждень; тест: підсумок містить ≥3 тренування або пояснює відсутність» — і довіряти агенту.
