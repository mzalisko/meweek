### 3. 🔴 Nested subagents: оптимізація витрат через маршрутизацію моделей
Нова практика з спільноти: встановити `CLAUDE_CODE_SUBAGENT_MODEL=haiku` на рівні shell, щоб всі leaf-субагенти за замовчуванням використовували Haiku замість Opus. Opus виправданий лише на рівні оркестратора (синтез). Leaf-агенти (читання файлів, grep, перевірка логів) — Haiku справляється за частку ціни.

- **Де**: [ofox.ai](https://ofox.ai/blog/claude-code-nested-subagents-2026/)
- 🟢 Ready to Apply | Impact: Medium | Effort: Low | Confidence: High

## 🤖 Claude Code та агенти

**Ключові інсайти тижня:**

1. **Nested subagents = ієрархія + розподіл за моделями**: Opus оркеструє, Haiku виконує leaf-задачі. Шаблон: `CLAUDE_CODE_SUBAGENT_MODEL=haiku` в shell env.
2. **Максимум 2–3 рівні вкладеності** на практиці, хоча ліміт — 5. Понад 3 рівні — токени ростуть, відлагоджувати важко.
3. **Session titles** тепер генеруються мовою розмови (v2.1.176) — корисно для мультимовних воркспейсів.
4. **Boris Cherny** керує десятками тисяч агентів одночасно; Claude Code — ~$2.5B annualized revenue, ~4% публічних комітів GitHub.

## Для Meweek

**1. Що варто спробувати сьогодні?**
Встановити `export CLAUDE_CODE_SUBAGENT_MODEL=claude-haiku-4-5-20251001` — одна команда в shell profile, миттєва економія на субагентних задачах без жодних змін у навичках.

**2. Яку нову навичку додати в `.claude/skills`?**
`subagent-cost-audit` — навичка, що аналізує поточні субагентні воркфлоу і пропонує оптимальну модель для кожного рівня (Opus/Sonnet/Haiku) на основі складності задачі.

**3. Яку рутину можна автоматизувати?**
Перевірку активації кредитів після billing split — невелика рутина раз на місяць перевіряє баланс кредитів через API і сповіщає про низький залишок.

**4. Який MCP сервер заслуговує уваги?**
`aaronsb/obsidian-mcp-plugin` — дає агентам traverse wikilink-граф Meweek замість сліпого пошуку файлів. Для сховища з сотнями нотаток це різниця між "знайти" і "знайти правильне".

**5. Яка одна ідея сьогодні дасть найбільшу економію часу?**
Прочитати Claude Code Guide 2026 (25 функцій, MarkTechPost) — ймовірно є 3–5 фіч, які вже доступні але не використовуються. Один раз 20 хвилин → постійна вигода.
