### 2. 🔴 Важлива зміна поведінки в Claude Code v2.1.186 (22 червня)
Bash-команди тепер автоматично тригерять відповідь Клода (раніше лише додавали контекст). Це може змінити поведінку автоматизованих сесій, де bash використовується для збору інформації без очікуваного reply.

- **Як відключити**: встановити `"respondToBashCommands": false` у `settings.json`.
- **Нові функції**: `claude mcp login <n>` / `claude mcp logout <n>` — аутентифікація MCP серверів прямо з CLI без `/mcp` меню. Підтримує `--no-browser` для SSH.
- 🟢 Ready to Apply | Impact: Medium | Effort: Low | Confidence: High

### 5. 🔴 Simon Willison: LLM не можуть відрізнити свій привілейований текст від user input

## Для Meweek

**1. Що варто спробувати сьогодні?**
Перевірити, чи не вплинув Claude outage (17:02–19:44 Київ) на будь-які scheduled рутини вчора. Якщо daily-digest не запустився — він зараз виконується. Якщо weekly-review планується — перевірити логи.

**2. Яку нову навичку додати в `.claude/skills`?**
`mcp-ssh-auth` — навичка для аутентифікації MCP серверів у headless/SSH середовищах через новий `claude mcp login --no-browser`, з збереженням токенів між сесіями.

**3. Яку рутину можна автоматизувати?**
Моніторинг Claude/API outages — невелика перевірка status.anthropic.com і сповіщення у разі інциденту перед запуском критичних рутин.

**4. Який MCP сервер заслуговує уваги?**
Офіційний MCP Registry (registry.modelcontextprotocol.io) — переглянути нові сервери у категоріях security та developer tools після оновлення до 5 000+.

**5. Яка одна ідея сьогодні дасть найбільшу економію часу?**
Додати `"respondToBashCommands": false` у відповідні навички — одна рядок у settings.json, яка прибирає зайві токени у всіх bash-heavy навичках.
