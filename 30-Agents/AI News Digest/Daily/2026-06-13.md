## 🔴 Найважливіше

### 1. 🔴 Claude Code v2.1.174–175 — керовані налаштування для команд
**v2.1.175**: новий managed-параметр `enforceAvailableModels` — коли увімкнено, список `availableModels` блокує і модель за замовчуванням (Default не може розв'язатись у заборонену модель, відбувається fallback на першу дозволену). Користувач або проєкт не можуть розширити список allowlist. Критично для enterprise-команд, що хочуть контролювати витрати.

**v2.1.174**: `/model picker` тепер явно показує, яку модель розв'язує Default (Opus на Max/Enterprise, Sonnet на Pro). Виправлено мерехтіння в JetBrains 2026.1+, dropped Shift+non-ASCII у Kitty keyboard protocol.

**Дія: якщо керуєш командою — вивчити `enforceAvailableModels` для обмеження моделей і бюджету.**

## 🔌 MCP та інтеграції

MCP spec RC — стабільна основа. Впроваджувати MCP-сервери вже безпечно: код не потребує переписування після фінального релізу. Stateless transport відкриває шлях до serverless MCP-серверів (AWS Lambda, Cloudflare Workers).

## Для Meweek

**1. Що варто спробувати сьогодні?**
Активувати Agent SDK кредити (дедлайн 15 червня) і протестувати оновлений `/model picker` — переконатись, що Default резолвиться в очікувану модель.

**2. Яку нову навичку додати в `.claude/skills`?**
`model-budget` — навичка-хелпер для вибору моделі за типом задачі (інтерактивна = Fable 5, автоматична = Sonnet, масова обробка = Haiku).

**3. Яку рутину можна автоматизувати?**
Щотижневий аудит використання Agent SDK кредитів — перевіряти залишок перед запуском важких рутин.

**4. Який MCP сервер заслуговує уваги?**
Serverless MCP на Cloudflare Workers — stateless spec робить це реальним без підтримки постійного сервера.

**5. Яка одна ідея сьогодні дасть найбільшу економію часу?**
Обмежити автоматичні `claude -p` задачі до Sonnet замість Fable 5 — зменшить витрату кредитів у 5×, дасть більше запусків за той самий $20/міс пул.
