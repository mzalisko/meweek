## 🔴 Найважливіше

### 1. 🔴 Claude Code v2.1.178: Tool(param:value) синтаксис дозволів
**Найважливіша практична новинка дня.** Тепер правила permissions підтримують синтаксис `Tool(param:value)` — дозволи на рівні параметрів інструменту, з wildcard `*`.

Приклади:
```json
{
  "permissions": {
    "deny": ["Agent(model:claude-fable-5)", "Bash(cmd:rm*)"],
    "allow": ["Agent(model:claude-haiku-4-5-20251001)"]
  }
}
```

- `Agent(model:opus)` — заблокувати Opus субагентів (дорогі задачі)
- `Bash(cmd:rm*)` — заблокувати деструктивні bash команди
- `Agent(model:*)` — дозволити/заблокувати всі субагентні виклики

- **Дія**: оновити `.claude/settings.json` рутин з обмеженнями по моделях.
- 🟢 Ready to Apply | Impact: High | Effort: Low | Confidence: High

## Для Meweek

**1. Що варто спробувати сьогодні?**
Додати `Tool(param:value)` restrictions у `.claude/settings.json` рутин: `"deny": ["Agent(model:claude-fable-5)"]` — одна зміна, яка захищає кредитний пул від дорогих випадкових викликів Fable.

**2. Яку нову навичку додати в `.claude/skills`?**
`permission-audit` — навичка, що читає `.claude/settings.json` і перевіряє: чи є обмеження по моделях у агентних викликах, чи немає hardcoded дорогих моделей, чи налаштовано `CLAUDE_CODE_SUBAGENT_MODEL`.

**3. Яку рутину можна автоматизувати?**
GitHub issue triage через GitHub Agentic Workflows — new issues у meweek автоматично отримують мітки і короткий аналіз від Claude.

**4. Який MCP сервер заслуговує уваги?**
Anthropic's MCP connector для GitHub Agentic Workflows — дозволяє Claude безпосередньо взаємодіяти з GitHub без PAT через `GITHUB_TOKEN`.

**5. Яка одна ідея сьогодні дасть найбільшу економію часу?**
Перевірити і активувати Anthropic Agent SDK кредити (якщо ще не зроблено) — без цього всі рутини з `claude -p` можуть мовчки зупинитись.
