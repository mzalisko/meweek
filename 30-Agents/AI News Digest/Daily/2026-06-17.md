## 🔴 Найважливіше

### 1. 🔴 Claude Code 2.1.181 — великий реліз (17 червня)

Найбільше оновлення за тиждень. Ключові зміни:

- **`/config key=value`** — встановлення будь-якого налаштування прямо з промпту (`/config thinking=false`)
- **`CLAUDE_CLIENT_PRESENCE_FILE`** — якщо файл існує, push-сповіщення на мобільний не надсилаються (зручно поруч з комп'ютером)
- **Streaming** — довгі абзаци тепер стрімляться рядок за рядком, а не чекають першого переносу рядка
- **Авто-повтор** — обриви API під час thinking автоматично ретраяться без втрати відповіді
- **Субагент-панель** — idle-агенти ховаються через 30 с, список обмежено 5 рядками зі скролом
- **Network drive fix** — виправлено Write/Edit, що створювали 0-байтні файли на мережевих дисках і cloud-синхронізованих папках
- **Prompt caching fix** — кешування тепер працює на `ANTHROPIC_BASE_URL` і Foundry

🟢 Ready to Apply | Impact: High | Effort: Low | Confidence: High

## 🤖 Claude Code та агенти

### Dynamic Workflows — нагадування (загальнодоступно з червня)

Dynamic Workflows у Claude Code (Opus 4.8) дозволяють:
- Сотні паралельних субагентів в одній сесії
- Автоматична генерація orchestration скрипту
- Верифікація результатів перед відповіддю
- Use cases: міграції сотень тисяч рядків, security audit, bug investigation

Доступно: Enterprise / Team / Max плани + API + Bedrock / Vertex / Foundry.

**Джерело**: [Anthropic Blog](https://claude.com/blog/introducing-dynamic-workflows-in-claude-code) · [InfoQ](https://www.infoq.com/news/2026/06/dynamic-workflows-claude-code/)

---

### Claude Design ↔ Claude Code інтеграція

Claude Design тепер може починати з локального коду і передавати макет у Claude Code. Claude Code отримав можливість редагувати дизайни прямо зі свого інтерфейсу. Це закриває gap між дизайном і реалізацією без ручного copy-paste.

## 🔌 MCP та інтеграції

### MCP Release Candidate — stateless core + HTTP load balancing

Наближається фінальний реліз (28 липня 2026):
- **Stateless core** — сервери масштабуються на звичайній round-robin HTTP-інфраструктурі
- **`Mcp-Method` header** — роутинг без інспекції тіла
- **`ttlMs` + `cacheScope`** — клієнти знають точно як довго кешувати список tools
- **MCP Apps** — server-rendered UI через MCP
- **Tasks extension** — long-running робота через MCP
- **OAuth/OpenID Connect** — переглянута авторизація

**Джерело**: [MCP Blog](https://blog.modelcontextprotocol.io/posts/2026-07-28-release-candidate/) · [The New Stack](https://thenewstack.io/model-context-protocol-roadmap-2026/)

## Для Meweek

**1. Що варто спробувати сьогодні?**
`/config thinking=false` для швидких задач і `CLAUDE_CLIENT_PRESENCE_FILE` для тиші сповіщень поки за комп'ютером.

**2. Яку нову навичку додати в `.claude/skills`?**
`dynamic-workflow-runner` — навичка яка допомагає скласти план паралельних субагент-задач для великих рефакторингів/міграцій і запускає Dynamic Workflow через Opus 4.8.

**3. Яку рутину можна автоматизувати?**
Моніторинг витрат на субагентів (як у Uber-кейсі): щоденна перевірка billing у console.anthropic.com + алерт якщо перевищено поріг.

**4. Який MCP сервер заслуговує уваги?**
Obsidian CLI (лютий 2026) — чистий інтерфейс для AI-агентів писати у vault. Потенційно краща альтернатива прямому fs-читанню через Claude Code.

**5. Яка одна ідея дасть найбільшу економію часу?**
**GLM-5.2 локально** для конфіденційних задач + **Dynamic Workflows** для великих міграцій коду — разом закривають gap між вартістю і масштабом задач.
