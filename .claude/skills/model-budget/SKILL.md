# model-budget — яка модель під яку задачу

Це скіл-**порадник**. Він нічого не запускає і нічого не змінює — лише радить, яку модель Claude взяти під конкретну задачу, і показує, наскільки одна дорожча за іншу. Рішення завжди за людиною.

## Коли застосовувати

- Перед тим, як створити або змінити хмарну рутину — який `model` їй прописати.
- Коли задача дорога або масова, і хочеться зекономити, не втративши якість.
- Коли треба зрозуміти, «де ми переплачуємо за надто потужну модель».

## Матриця і ціни — спільне джерело

Канонічна матриця вибору й таблиця цін живуть у `50-Guides/Гайд — Навички Meweek простими словами.md` (розділ про `/model-budget`) — спільні для всіх агентів, єдине джерело без дублювання. Звідти бери рекомендацію й відносну вартість; нижче — лише як це застосувати.

Ціни там — орієнтир станом на 2026-06 (API-прайс за вихідні токени, за яким тарифікуються `claude -p` / Agent SDK після 15.06.2026). Перед фінансовими рішеннями звір актуальні ціни через навичку `claude-api` або `platform.claude.com/pricing`.

## Як відповідати на `/model-budget <опис задачі>`

1. Визнач тип задачі за матрицею.
2. Назви рекомендовану модель і одне речення чому.
3. Дай орієнтовну відносну вартість (× від Haiku, або × від моделі, якою користувач робить це зараз).
4. Якщо задача не відповідає моделі (напр. складний аналіз на Haiku або шаблонний звіт на Fable) — попередь.
5. Якщо це для рутини — підкажи готовий рядок: `"model": "claude-..."`.
6. Без аргументів — просто покажи матрицю вибору.

## Обслуговування

Головна цінність — **матриця** (вона стабільна), ціни вторинні й застарівають. Оновлювати при зміні лінійки моделей. Зміни вносити через `skill-forge` (режим самопокращення), щоб зафіксувати їх у журналі.
