# model-budget — яка модель під яку задачу

Це скіл-**порадник**. Він нічого не запускає і нічого не змінює — лише радить, яку модель Claude взяти під конкретну задачу, і показує, наскільки одна дорожча за іншу. Рішення завжди за людиною.

## Коли застосовувати

- Перед тим, як створити або змінити хмарну рутину — який `model` їй прописати.
- Коли задача дорога або масова, і хочеться зекономити, не втративши якість.
- Коли треба зрозуміти, «де ми переплачуємо за надто потужну модель».

## Матриця вибору

| Тип задачі | Модель | Чому |
|---|---|---|
| Інтерактивна розробка, складна архітектура, критичний код (DBManager) | `claude-fable-5` або `claude-opus-4-8` | Максимальна якість; ціна виправдана там, де помилка дорога |
| Автоматичні рутини (дайджест, тижневий огляд) | `claude-sonnet-4-6` | Якість достатня, ~1.7× дешевше за Opus і ~3.3× за Fable |
| Шаблонна генерація (Fitness-звіти) | `claude-haiku-4-5` | Структурований вивід, мінімум міркувань — найшвидше й найдешевше |
| Масова проста обробка, класифікація | `claude-haiku-4-5` | Найдешевше за токен |
| Code review / security review | `claude-sonnet-4-6` (для DBManager-безпеки — підняти до `claude-opus-4-8`/`claude-fable-5`) | Баланс якості й вартості; критичне варте дорожчої моделі |

## Ціни (орієнтир, output — головний драйвер вартості)

| Модель | $/1M output | Відносно Haiku |
|---|---|---|
| `claude-haiku-4-5` | $5 | 1× |
| `claude-sonnet-4-6` | $15 | 3× |
| `claude-opus-4-8` | $25 | 5× |
| `claude-fable-5` | $50 | 10× |

Дані станом на 2026-06. Це API-прайс за вихідні токени — саме за ним тарифікуються `claude -p` та Agent SDK після 15.06.2026. Вхідні токени дешевші (приблизно вп'ятеро), але вивід зазвичай і визначає рахунок. Перед фінансовими рішеннями звір актуальні ціни (навичка `claude-api` або `platform.claude.com/pricing`) — лінійка й тарифи змінюються.

## Як відповідати на `/model-budget <опис задачі>`

1. Визнач тип задачі за матрицею.
2. Назви рекомендовану модель і одне речення чому.
3. Дай орієнтовну відносну вартість (× від Haiku, або × від моделі, якою користувач робить це зараз).
4. Якщо задача не відповідає моделі (напр. складний аналіз на Haiku або шаблонний звіт на Fable) — попередь.
5. Якщо це для рутини — підкажи готовий рядок: `"model": "claude-..."`.
6. Без аргументів — просто покажи матрицю вибору.

## Обслуговування

Головна цінність — **матриця** (вона стабільна), ціни вторинні й застарівають. Оновлювати при зміні лінійки моделей. Зміни вносити через `skill-forge` (режим самопокращення), щоб зафіксувати їх у журналі.
