# Новий проєкт

## Кроки

1. Уточни в користувача (якщо не сказано): назву, мету одним реченням, чи буде код у цій папці чи окремим репозиторієм.
2. Створи папку `10-Проєкти/<Назва>/` з файлами:
   - `<Назва> — Огляд.md` — дашборд за `_Шаблони/Шаблон проєкту.md` (мета, статус, ключові файли, журнал)
   - `<Назва> — Вимоги.md` — функціональні/нефункціональні вимоги + розділ «Питання для уточнення» з чекбоксами
   - `<Назва> — План.md` — план етапами (Етап 0 — уточнення, далі прототип → стабілізація → продакшн), кожен пункт — чекбокс
   - `<Назва> — Контекст.md` — знімок стану для агентів (≤ 30 рядків): стан, останні рішення, наступний крок
3. Додай проєкт у таблицю «Активні» в `10-Проєкти/Проєкти.md` і в список у `Старт.md`.
4. Якщо проєкт із підвищеними вимогами до безпеки — додай `<Назва> — Безпека.md` (модель загроз, як у NMDB).
5. Якщо буде код — створи `10-Проєкти/<Назва>/код/` із власним `AGENTS.md` (правила саме для коду цього проєкту).
6. Заверши комітом: `<назва>: створено проєкт`.

## Правила

- Усі файли українською. Назви файлів — `<Назва> — <Тип>.md` (em dash).
- Огляд — це індекс проєкту: всі інші файли мають бути в ньому перелічені (принцип «індекс на кожному рівні»).
- Не дублюй інформацію між файлами — посилання `[[...]]` замість копіювання.
