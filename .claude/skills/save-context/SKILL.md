# Збереження контексту

## Кроки

1. Визнач, над якими проєктами йшла робота в сесії.
2. Для кожного онови `10-Проєкти/<Назва>/<Назва> — Контекст.md`:
   - **Оновлено:** сьогоднішня дата
   - **Стан:** поточний етап плану
   - **Останні рішення:** 2–3 пункти з цієї сесії
   - **Наступний крок:** один конкретний пункт
   - **Відкриті питання:** що блокує
3. Тримай файл ≤ 30 рядків — це знімок, не журнал (історія є в git).
4. Прийняті технічні рішення оформи як ADR у папці проєкту (`_Шаблони/Шаблон ADR.md`), додай посилання в Огляд.
5. Відміть виконані чекбокси в `<Назва> — План.md`.
6. Закоміть: `<назва>: оновлено контекст сесії`.

## Чому так

Чат одноразовий, файли постійні. Після /clear новій сесії достатньо прочитати лише файл контексту — це економить токени і нічого не губиться.
