# Гайд — Навички Meweek простими словами

Тут зібрані всі наші навички (skills) і слеш-команди — пояснені по-людськи: **що воно робить, коли його кликати, як покликати**. Це жива шпаргалка: кожна нова навичка одразу дописується сюди (це робить `skill-forge`). Якщо навички немає в цьому списку — вважай, що її «не існує», бо ніхто не знатиме, що вона є.

**Цей файл спільний для всіх агентів** (Claude, Codex, Gemini). Кожен звіряється з ним перед роботою, щоб не дублювати вже наявне, і дописує сюди кожну нову можливість. Це і є те, що тримає всіх агентів «як одне ціле»: правила тут, а не в конфігу окремого агента.

> Навичка — це готовий сценарій, який агент виконує за тебе. Команда `/назва` — це просто кнопка, що запускає сценарій. Той самий сценарій працює і в Codex/Gemini (там без слеша).

## Щодня й щосесії

### `/sync` — синхронізація
**Що це.** Підтягує свіже з GitHub і/або зберігає твою роботу туди. Заодно вливає результати хмарних рутин (вони приходять окремими гілками).
**Коли.** На початку роботи (підтягнути) і в кінці (зберегти й запушити).
**Як.** `/sync`
**Приклад.** Сів за роботу → `/sync` → маєш найсвіжіший стан.

### `/save-context` — зберегти пам'ять проєкту
**Що це.** Стискає поточний стан проєкту в короткий файл `<Проєкт> — Контекст.md`, щоб наступна сесія почала не з нуля.
**Коли.** Перед `/clear`, перед зміною теми, наприкінці сесії.
**Як.** `/save-context`
**Приклад.** Награвся з DBManager, ідеш спати → `/save-context` → завтра агент одразу в курсі.

## Проєкти

### `/new-project` — завести проєкт
**Що це.** Створює нову папку проєкту з усіма потрібними файлами (огляд, контекст) за шаблоном.
**Коли.** З'явилась ідея, яку треба оформити як проєкт.
**Як.** `/new-project` або просто «створи проєкт <назва>».

### `/fitness` — фітнес-проєкт
**Що це.** Веде фітнес-щоденники (Микола, Марта): додає вагу, їжу, підбиває тиждень, імпортує звіти.
**Коли.** Щоденні записи й тижневі підсумки по фітнесу.
**Як.** `/fitness Микола`, `/fitness Марта`, `/fitness Микола підбий тиждень`.

### `docker-project` — оточення в Docker
**Що це.** Піднімає Docker під проєкт (бази, сервіси для розробки).
**Коли.** Проєкту потрібні БД/сервіси локально.
**Як.** «контейнеризуй <проєкт>» або «підніми оточення».

## Обслуговування сховища

### `/weekly-review` — розбір сховища
**Що це.** Раз на тиждень наводить лад: розбирає вхідні, перевіряє контексти проєктів, шукає биті посилання, пропонує що в архів.
**Коли.** Раз на тиждень або коли в сховищі бардак.
**Як.** `/weekly-review`

### `/review` — критичне рев'ю змін
**Що це.** Дивиться на твої незакомічені зміни очима незалежного рев'ювера й каже, що не так.
**Коли.** Перед комітом важливого (особливо критичний код — DBManager, безпека).
**Як.** `/review`

## Гроші та якість моделей

### `/model-budget` — яка модель під яку задачу
**Що це.** Радить, якою моделлю Claude робити конкретну задачу, і показує, наскільки одна дорожча за іншу. Нічого не запускає — лише радить.
**Коли.** Перед налаштуванням рутини; коли задача масова й хочеться зекономити; для аудиту «де переплачуємо».
**Як.** `/model-budget <опис задачі>` (без аргументів — покаже таблицю).
**Приклад.** `/model-budget щоденний дайджест новин` → «бери `claude-sonnet-4-6`, у ~3.3 раза дешевше за Fable, якості вистачить».

#### Матриця вибору моделі (спільна для всіх агентів)

Це канонічне джерело — і Claude через `/model-budget`, і Codex/Gemini читають саме цю таблицю.

| Тип задачі | Модель | Чому |
|---|---|---|
| Інтерактивна розробка, складна архітектура, критичний код (DBManager) | `claude-fable-5` або `claude-opus-4-8` | Максимальна якість; ціна виправдана там, де помилка дорога |
| Автоматичні рутини (дайджест, тижневий огляд) | `claude-sonnet-4-6` | Якості достатньо, ~1.7× дешевше за Opus і ~3.3× за Fable |
| Шаблонна генерація (Fitness-звіти) | `claude-haiku-4-5` | Структурований вивід, мінімум міркувань — найшвидше й найдешевше |
| Масова проста обробка, класифікація | `claude-haiku-4-5` | Найдешевше за токен |
| Code review / security review | `claude-sonnet-4-6`; для DBManager-безпеки — `claude-opus-4-8`/`claude-fable-5` | Баланс; критичне варте дорожчої моделі |

Ціни (output, головний драйвер рахунку; орієнтир 2026-06):

| Модель | $/1M output | Відносно Haiku |
|---|---|---|
| `claude-haiku-4-5` | $5 | 1× |
| `claude-sonnet-4-6` | $15 | 3× |
| `claude-opus-4-8` | $25 | 5× |
| `claude-fable-5` | $50 | 10× |

Це API-прайс за вихідні токени — за ним тарифікуються `claude -p` / Agent SDK після 15.06.2026. Перед фінансовими рішеннями звір актуальні ціни (навичка `claude-api` або `platform.claude.com/pricing`) — лінійка й тарифи змінюються.

## Розвиток самих навичок

### `/skill-forge` — кузня навичок
**Що це.** Перетворює ідею зі `30-Agents/Skill Ideas/` на готову навичку (і дописує її в цей гайд), а ще вміє покращувати вже наявні навички.
**Коли.** «впровадь ідею <…>», «зроби скіл з <…>», «покращ навичку <…>», «розвинь скіли».
**Як.** `/skill-forge впровадь model-budget` або `/skill-forge покращ git-sync`.
**Приклад.** Хмарна рутина накидала ідей у Skill Ideas → `/skill-forge впровадь <ідея>` → варте стає робочою навичкою з поясненням тут.
