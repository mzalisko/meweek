# Гайд — Інтеграція з CRM та автоматичне заповнення даних

Цей гайд описує, як налаштувати взаємодію між вашою CRM-системою та **DBManager Core** для автоматичного додавання номерів телефонів, оновлення динамічних значень та керування гарячим перемиканням ліній (failover) при збоях.

---

## Варіант 1: Автоматичне перемикання номерів при збоях у CRM (Webhook API)

Це основний спосіб інтеграції. Якщо лінія в CRM зайнята, заблокована або не відповідає, CRM може надіслати запит на Core, і сайт протягом 1 хвилини автоматично перемкнеться на резервний номер.

### 1. Адреса ендпоінту
`POST https://core.yourdomain.com/api/monitoring/numbers`

### 2. Заголовки запиту
```http
Content-Type: application/json
X-Signature: <HMAC-SHA256-підпис-тіла-запиту>
```

### 3. Схема запиту (JSON payload)
```json
{
  "e164": "+380501234567",
  "status": "down"
}
```
* **`e164`** (string, обов'язкове): номер телефону у міжнародному форматі E.164, який має бути змінено.
* **`status`** (string, обов'язкове): новий статус лінії. Дозволені значення:
  * `down` — лінія не працює (сайт миттєво перемкнеться на резервний номер із ланцюга).
  * `active` — лінію відновлено (сайт повернеться на основний номер або залишиться на резерві залежно від режиму Sticky).

### 4. Генерація підпису X-Signature
Запит захищено підписом HMAC-SHA256 для запобігання несанкціонованим перемиканням.
1. Візьміть сирий JSON-рядок запиту (наприклад, `{"e164":"+380501234567","status":"down"}`).
2. Обчисліть HMAC-SHA256 від цього рядка, використовуючи секрет `MONITORING_WEBHOOK_SECRET` із файлу `.env` вашого Core.
3. Передайте отриманий хеш у заголовку `X-Signature`.

## Варіант 2: Пряме заповнення бази даних з CRM (Direct DB Import)

Оскільки Core розроблений на Laravel і використовує реляційну базу даних MySQL, CRM-система може імпортувати номери телефонів та створювати слоти безпосередньо через SQL-запити.

### Схема зв'язків таблиць для телефонів:
```
┌─────────────────┐       ┌─────────────────┐       ┌──────────────────┐
│  data_values    │ ◄──── │   phone_slots   │ ◄──── │  number_entries  │
│  (ключ шорткоду)│       │  (налаштування) │       │  (пріоритети)    │
└─────────────────┘       └─────────────────┘       └────────┬─────────┘
                                                             │
                                                             ▼
                                                    ┌──────────────────┐
                                                    │  phone_numbers   │
                                                    │  (фізичні номери)│
                                                    └──────────────────┘
```

### Крок 1: Додавання фізичного номера телефону
Вставте новий номер телефону в таблицю `phone_numbers`:
```sql
INSERT INTO phone_numbers (e164, label, status, created_at, updated_at) 
VALUES ('+380501234567', 'CRM Основной UA', 'active', NOW(), NOW());
```

### Крок 2: Створення логічного значення (слота)
Вставте запис у таблицю `data_values` (де визначається ключ шорткоду):
```sql
INSERT INTO data_values (value_type_id, `key`, scope_type, scope_id, status, created_at, updated_at) 
VALUES (
  1,               -- ID типу "phone" (можна звірити в таблиці value_types)
  'phone_ua_1',    -- Ключ, який вставляється в шорткод [dbm key="phone_ua_1"]
  'site',          -- Область видимості (завжди 'site')
  12,              -- ID сайту, до якого належить телефон (таблиця sites)
  'active', 
  NOW(), 
  NOW()
);
```

### Крок 3: Налаштування політики слота
Створіть запис у таблицю `phone_slots`, пов'язавши його з створеним `data_value_id`:
```sql
INSERT INTO phone_slots (data_value_id, return_mode, exhaustion_policy, emergency_number, created_at, updated_at)
VALUES (
  LAST_INSERT_ID(), -- ID щойно створеного data_values
  'auto',           -- Режим повернення ('auto' або 'sticky')
  'hide',           -- Що робити при вичерпанні ліній ('hide' - сховати, 'show_last')
  NULL,             -- Аварійний номер, якщо всі лінії впали
  NOW(),
  NOW()
);
```

### Крок 4: Прив'язка номера до слота з пріоритетом
Створіть зв'язок у таблиці `number_entries` (визначає чергу резервування):
```sql
INSERT INTO number_entries (phone_slot_id, phone_number_id, priority, created_at, updated_at)
VALUES (
  15, -- ID створеного phone_slots
  1,  -- ID створеного phone_numbers
  0,  -- Пріоритет (0 — основний, 1, 2... — резервні)
  NOW(),
  NOW()
);
```

> [!IMPORTANT]
> Після прямого оновлення або додавання даних у базу Core, зміни автоматично не поширюються на сайти WordPress.
> Щоб дані доставилися на сайти, Core має згенерувати payload. Для цього ви можете викликати вбудований метод публікації сайту через консольну команду Laravel або натиснути кнопку "Опублікувати" в адмінці Core.
