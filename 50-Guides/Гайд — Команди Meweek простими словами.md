# Гайд — Команди Meweek простими словами

Це коротка шпаргалка: які команди можна просити в агентів у Meweek і що очікувати на виході. У Claude більшість команд запускається зі слешем (`/sync`), у Codex/Gemini частина проєктних команд пишеться без слеша (`fitness Микола`).

## Команди

| Команда | Коли використовувати | Результат |
|---|---|---|
| `/sync` | На старті або в кінці роботи | Підтягує/зберігає зміни через git |
| `/save-context` | Перед `/clear`, паузою або зміною теми | Оновлений короткий контекст проєкту |
| `/new-project <назва>` | Коли ідея стає окремим проєктом | Папка в `10-Projects/` з базовими файлами |
| `/weekly-review` | Коли треба навести лад у vault | Розбір inbox, контекстів, посилань і архіву |
| `/review` | Перед важливим комітом або критичним кодом | Список ризиків і помилок у поточних змінах |
| `/model-budget <задача>` | Коли треба вибрати модель Claude під задачу | Рекомендація моделі й орієнтир вартості |
| `/skill-forge впровадь <ідея>` | Коли ідея з `Skill Ideas` має стати навичкою | Навичка + команда + запис у людському гайді |
| `/fitness <учасник> [режим]` | У Claude для Fitness | Запис у журнал, уточнення, тижневий підсумок |
| `fitness <учасник> [режим]` | У Codex/Gemini для Fitness | Те саме, але без слеша |

## Як перевірити результат

1. Після команди агент має назвати змінені файли або сказати, що нічого не змінював.
2. Якщо команда створила або змінила щось постійне, має бути коміт.
3. Для нової команди або навички має оновитися [[Гайд — Навички Meweek простими словами]].
4. Для проєктної команди має оновитися відповідний файл `— Команди.md` або `— Як працювати.md`.

## Що просити агента

```text
Покажи доступні команди Meweek і коли яку використовувати.
```

```text
Створи команду /<назва>, але онови людський гайд і агентний контракт перед комітом.
```
