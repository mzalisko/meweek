# Гайд — Як підключити та налаштувати плагін DBManager

Цей гайд призначено для контент-менеджерів, розробників та адміністраторів сайтів. Він детально описує процес підключення плагіна **DBManager** до нового сайту WordPress, його налаштування та використання динамічних даних.

---

## 1. Встановлення плагіна

Плагін DBManager складається з двох частин: основного плагіна та аварійного (Must-Use fallback) плагіна для забезпечення 100% працездатності сайту, навіть якщо основний плагін випадково вимкнули.

### Крок 1: Встановлення основного плагіна
1. Скопіюйте папку `dbmanager` до директорії плагінів вашого сайту: `wp-content/plugins/`.
2. Перейдіть в адмін-панель WordPress -> **Плагіни**.
3. Знайдіть **DBManager** та натисніть **Активувати**.

### Крок 2: Встановлення аварійного плагіна (Must-Use)
1. Скопіюйте файл `dbmanager-fallback.php` та супровідний файл `render-core.php` з папки `mu/` у директорію Must-Use плагінів вашого сайту: `wp-content/mu-plugins/` (якщо папки `mu-plugins` немає, створіть її).
2. Цей крок є обов'язковим, оскільки при вимкненні чи видаленні основного плагіна цей fallback-плагін продовжує рендерити останні успішно закешовані дані замість порожніх шорткодів.

---

## 2. Налаштування зв'язку з DataBridge

Після активації плагіна в лівому меню адмін-панелі WordPress з'явиться розділ **DBManager**.

1. Перейдіть до **DBManager** -> **Налаштування**.
2. Заповніть обов'язкові поля зв'язку з вашим сервером DataBridge:
   - **URL DataBridge**: Адреса вашого проксі-сервера DataBridge (наприклад, `https://bridge.domain.com`).
   - **Токен сайта**: Унікальний ідентифікатор доступу для цього сайту.
   - **Секрет підпису даних**: Спільний HMAC-ключ для перевірки цілісності завантажених даних.
   - **Секрет пінга**: Спільний HMAC-ключ для автентифікації вхідних вебхуків від DataBridge.
   - **Назва шорткода (нейтральна)**: Назва шорткода, яка буде використовуватися на сайті (наприклад, `company_data` або просто `dbm`).
   - **CSS-клас виводу**: Опціональний CSS клас для стилізації обгорток значень.
3. Натисніть **Зберегти налаштування**.

---

## 3. Налаштування автоматичного оновлення (Крон)

Плагін автоматично реєструє дві регулярні задачі у WordPress Cron:
1. `dbm_daily_reconcile` (**щодобова**): плагін звертається до DataBridge з HTTP-заголовком `If-None-Match`, щоб звірити версію кешу. Якщо версії збігаються, трафік не витрачається.
2. `dbm_geodb_sync` (**щотижнева**): плагін автоматично оновлює локальну базу даних визначення країн MaxMind (`.mmdb`), стягуючи свіжу версію з DataBridge.

Переконайтеся, що на вашому хостингу налаштований системний крон для WordPress (рекомендується додавати `wp-cron.php` у системний `crontab` кожні 5-10 хвилин).

---

## 4. Використання шорткодів та PHP-коду

Усі дані, отримані з Core, можна динамічно вставляти в будь-яких частинах вашого сайту. Приклади для кожного ключа автоматично генеруються в розділі **DBManager** -> **Вставка**.

### A. Стандартний шорткод (у контенті сторінки, віджетах, заголовках)
Використовуйте шорткод з нейтральним ім'ям, налаштованим раніше (наприклад, `dbm`):
```text
Зателефонуйте нам: [dbm key="phone_1"]
```

Завдяки вбудованим фільтрам шорткоди автоматично виконуються у:
- Вмісті сторінок/записів (`the_content`)
- Заголовках сторінок (`the_title`)
- Уривках (`the_excerpt`)
- Стандартних текстових віджетах (`widget_text`)

### B. В ACF полях (Advanced Custom Fields)
Ви можете просто вставляти шорткод `[dbm key="phone_1"]` прямо в текстові поля, текстові області чи WYSIWYG редактори ACF. Плагін автоматично опрацює та розкриє шорткод перед виведенням поля на фронтенді.

### C. Використання безпосередньо в PHP шаблонах теми
Якщо ви розробляєте власну тему, ви можете вивести значення через глобальну PHP функцію:
```php
<?php echo dbm_get('phone_1'); ?>
```

## 5. Тестування локалізації (Геосимуляція)

Якщо у вас є ціни або номери телефонів, які мають показуватися лише відвідувачам із конкретних країн (наприклад, тільки для Румунії `RO` чи України `UA`), ви можете перевірити коректність відображення без використання VPN.

1. Перейдіть до **DBManager** -> **Геосимуляція**.
2. В таблиці "Поточний стан локалізації" ви побачите вашу реальну країну, визначену за IP, та активну країну.
3. Оберіть потрібну країну у випадаючому списку (наприклад, `RO`) та натисніть **Зберегти налаштування**.
4. Після цього сайт думатиме, що ви зайшли з обраної країни. Перейдіть на фронтенд та перевірте, чи правильні номери та ціни відображаються.
5. Після завершення тестування оберіть пункт `— Вимкнути симуляцію (визначати за IP) —` та збережіть налаштування знову.

---

## 6. Аварійна стійкість

- **Сайт ніколи не покаже порожнечу**: Дані надійно кешуються в базі даних сайту (`wp_options`). У разі недоступності DataBridge, сайт продовжуватиме відображати останні успішно завантажені дані.
- **Стійкість до вимкнення плагіна**: Якщо плагін буде випадково деактивовано, Must-Use fallback плагін перехопить шорткоди та відобразить дані з кешу для скоупу `WORLD`.
