# Гайд — Як підключити та налаштувати плагін DBManager

Цей гайд призначено для контент-менеджерів, розробників та адміністраторів сайтів. Він детально описує процес підключення плагіна **DBManager** до нового сайту WordPress, його налаштування та використання динамічних даних.

---

## 1. Встановлення плагіна

Плагін DBManager розроблений з урахуванням 100% працездатності сайту. Він автоматично встановлює та конфігурує аварійний фолбек-компонент.

### Крок 1: Встановлення
1. Скопіюйте папку `dbmanager` до директорії плагінів вашого сайту: `wp-content/plugins/`.
2. Перейдіть в адмін-панель WordPress -> **Плагіни**.
3. Знайдіть **DBManager** та натисніть **Активувати**.

### Крок 2: Автоматичне налаштування Fallback (Must-Use)
* Під час активації основного плагіна система **автоматично** створить директорію `wp-content/mu-plugins/` та скопіює туди аварійний плагін-фолбек.
* Цей механізм гарантує, що навіть якщо основний плагін випадково деактивують або видалять в адмінці, шорткоди на сайті не зламаються і продовжуватимуть виводити останні успішно збережені дані з кешу.

---

## 2. Налаштування та підключення токена

Після активації плагіна в лівому меню адмін-панелі WordPress з'явиться розділ **DBManager**.

1. Перейдіть до **DBManager** -> **Настройки**.
2. Вкажіть параметри підключення:
   - **Токен / Секрет подписи данных**: Унікальний токен вашого сайту, виданий CRM чи Core-панеллю (наприклад, `DWDrpX1RsqBMkwAGssOZO4CrOKy25cqUnjTC7qtacOsu49Yz`). Він використовується для безпечної верифікації вхідного HMAC-підпису.
   - **Название шорткода (нейтральное)**: Нейтральне ім'я для використання на сторінках (наприклад, `dbm` або `company_data`).
   - **CSS-класс вывода**: Опціональний клас для стилізації обгортки значень.
3. Натисніть **Сохранить настройки**.

---

## 3. Безпека: Повна конфіденційність (Pure Push-only)

Плагін спроєктований так, щоб ніхто не міг відстежити адресу CRM чи центральної панелі:
- **Нуль вихідних запитів**: Плагін є пасивним отримувачем. Він ніколи не ініціює HTTP-з'єднання з сайту до CRM. У логах сервера вашого сайту немає жодної згадки про CRM.
- **Односторонній Push**: CRM (або Core) самостійно відправляє зашифровані/підписані дані на REST API сайту: `POST https://your-site.com/wp-json/dbm/v1/ping`. Плагін перевіряє підпис токеном та оновлює кеш.

---

## 4. Використання шорткодів та PHP-коду

Усі дані, отримані з CRM, можна динамічно вставляти в будь-яких частинах вашого сайту. Приклади для кожного ключа автоматично генеруються в розділі **DBManager** -> **Вставка**.

### A. Стандартний шорткод (у контенті сторінки, віджетах, заголовках)
Використовуйте шорткод з нейтральним ім'ям, налаштованим раніше (наприклад, `dbm`):
```text
Зателефонуйте нам: [dbm key="phone_1"]
```

Завдяки вбудованим фільтрам шорткоди автоматично виконуються у:
- Вмісті сторінок/записів (`the_content`)
- Заголовках сторінок (`the_title`)
- Уривках (`the_excerpt`)
- Стандартних текстових віджетах (`widget_text`)

### B. В ACF полях (Advanced Custom Fields)
Ви можете вставляти шорткод `[dbm key="phone_1"]` безпосередньо в будь-які текстові або WYSIWYG поля ACF. Плагін автоматично опрацює та розкриє шорткод перед виведенням поля на фронтенді.

### C. Використання безпосередньо в PHP шаблонах теми
Якщо ви розробляєте власну тему, ви можете вивести значення через глобальну PHP функцію:
```php
<?php echo dbm_get('phone_1'); ?>
```

---

## 5. Тестування локалізації (Геосимуляція)

Якщо у вас є номери телефонів чи ціни, які мають показуватися лише відвідувачам із конкретних країн (наприклад, тільки для Румунії `RO` чи України `UA`), ви можете перевірити коректність відображення без використання VPN.

1. Перейдіть до **DBManager** -> **Геосимуляция**.
2. Оберіть потрібну країну у випадаючому списку (базові країни **UA, RO, RU, BY, KZ, PL** доступні завжди) та натисніть **Сохранить настройки**.
3. Після цього сайт думатиме, що ви зайшли з обраної країни. Перейдіть на фронтенд та перевірте, чи правильні номери та ціни відображаються.
4. Після завершення тестування оберіть пункт `— Выключить симуляцию (определять по IP) —` та збережіть налаштування знову.
