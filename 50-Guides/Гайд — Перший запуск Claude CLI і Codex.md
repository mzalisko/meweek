# Гайд — Перший запуск Claude CLI і Codex

Як запустити термінальних агентів у `C:\Dev\Meweek` вперше і не отримати конфлікти.

## Claude Code (перший запуск)

```powershell
cd C:\Dev\Meweek
claude
```

1. Перший раз: логін через браузер (акаунт Anthropic).
2. Claude автоматично читає `CLAUDE.md` → `AGENTS.md` — правила середовища вже діють.
3. Перевір: спитай «які в тебе правила в цій папці?» — має переказати AGENTS.md.
4. Перевір команди: набери `/` — у списку мають бути `/new-project`, `/save-context`, `/weekly-review`.
5. Далі (опційно): встанови superpowers ([[Гайд — Superpowers]]) і GitHub MCP ([[Гайд — Git і GitHub MCP]]).

## Codex CLI (перший запуск)

```powershell
npm install -g @openai/codex
cd C:\Dev\Meweek
codex
```

1. Перший раз: логін (акаунт OpenAI).
2. Codex автоматично читає `AGENTS.md` з кореня.
3. Рекомендований режим — **auto** (читає/пише файли, питає на ризиковані дії).
4. Перевір: «які правила цього середовища?».

## Як агенти не конфліктують (важливо)

Принцип: **спільна пам'ять — це файли + git, а не чат**. Правила зашиті в `AGENTS.md`, розділ «Співіснування агентів»:

1. **Один агент за раз** на одну задачу. Не запускати Claude і Codex одночасно над одним проєктом.
2. **Перед роботою** агент робить `git status` — якщо є чужі незакомічені зміни, питає тебе, а не перезаписує.
3. **Після роботи** — оновлення `<Проєкт> — Контекст.md` + коміт. Саме так наступний агент (будь-який) дізнається стан: `git log` + файл контексту.
4. **Конфіги розділені**: `.claude/` — лише Claude, `.codex/` — лише Codex, `.gemini/` — лише Gemini. Агенти не редагують чужі конфіги.
5. **Спільні правила — лише в `AGENTS.md`**: змінив правило один раз — його бачать усі агенти. Не дублювати правила в CLAUDE.md/GEMINI.md (там лише вказівники).

## Типовий робочий цикл

```
запустив агента → «прочитай контекст NMDB» → працюєте → «збережи контекст» → /clear або вихід
```

Наступна сесія (будь-якого агента) починається з читання того ж контексту — нічого не губиться.
