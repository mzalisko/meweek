# Гайд — Obsidian для Meweek

Цей гайд пояснює, як вести сховище [[Старт|Meweek]] в Obsidian так, щоб нотатки були корисні людині, агентам і git-історії.

## Базова ідея

Obsidian тут не просто редактор нотаток. Це робоча база знань, де:

- поточний стан проєктів живе у файлах `10-Projects/<Назва>/<Назва> — Контекст.md`;
- довші рішення живуть у проєктних нотатках або ADR;
- гайди живуть у [[Гайди|50-Guides]];
- агенти читають тільки потрібний контекст, а не все сховище.

## Щоденна робота

1. Починати з [[Старт]] або огляду потрібного проєкту.
2. Для нової ідеї без місця створювати нотатку в `00-Inbox`.
3. Для проєктної роботи створювати або оновлювати нотатку в `10-Projects/<Назва>/`.
4. Важливі рішення одразу переносити з чату в файл.
5. Перед завершенням роботи оновлювати `<Проєкт> — Контекст.md`.

## Назви нотаток

- Назви `.md` файлів — українською.
- Проєктні файли: `<Проєкт> — <Тип>.md`, наприклад `Fitness — Меню.md`.
- Індекси розділів: коротко й очевидно, наприклад `Гайди.md`, `Проєкти.md`.
- Не створювати нотатки в корені без причини.

## Посилання

1. Зв'язувати нотатки через `[[wiki-посилання]]`.
2. Кожна проєктна нотатка має посилатися на `<Проєкт> — Огляд` або іншу центральну нотатку проєкту.
3. Не дублювати один і той самий факт у кількох місцях: краще дати посилання.
4. Для таблиць-індексів використовувати посилання на нотатки, а не просто текстові назви.

## Граф

Граф Obsidian будується автоматично з посилань між нотатками. Він не є ручною схемою, тому глобальний граф може виглядати хаотично.

Для читабельності:

- дивитися Local Graph від центральної нотатки проєкту;
- для одного проєкту використовувати фільтр `path:10-Projects/Fitness`;
- для гайдів використовувати `path:50-Guides`;
- для агентів використовувати `path:30-Agents`;
- приховувати зайве через фільтри, а не редагувати `.obsidian/` без потреби.

## Як робити граф кращим

1. У кожного проєкту має бути центральна нотатка `<Проєкт> — Огляд`.
2. Усі дочірні нотатки проєкту мають посилатися на огляд.
3. Контекст, план, вимоги, архітектура, меню, журнали — окремі вузли, не один великий файл.
4. Сирітські нотатки або пов'язати, або перенести в `00-Inbox` до розбору.
5. Теги використовувати для категорій, а не як заміну посиланням.

## Робота з агентами

Коли просиш агента працювати з Obsidian:

```text
Прочитай контекст <НазваПроєкту> і продовжуй роботу тільки в цій папці.
```

Для оновлення стану:

```text
Збережи контекст <НазваПроєкту>: коротко онови стан, останні рішення, наступний крок і відкриті питання.
```

Для нового гайду:

```text
Додай гайд у 50-Guides, українською, з посиланням з [[Гайди]].
```

## Перевірка перед комітом

1. `git status` — переконатися, що в коміт входять тільки потрібні файли.
2. Перевірити, що нові нотатки мають посилання на центральні нотатки.
3. Не комітити `.obsidian/graph.json`, якщо зміна не була навмисною.
4. Коміт українською у форматі `розділ: що зроблено`.
