# Гайд — Адмінка DBManager (CRM)

Внутрішня адмінка центру DBManager Core. Технічна спека UI — [[DBManager — Дизайн]] §7; повна картина — `10-Projects/DBManager/documentation/`.

## Що це

CRM-адмінка, де керують контактними даними парку сайтів: дивляться значення (телефони, месенджери, ціни…), їхній стан failover, область дії (групове / перекриття сайта). Внутрішня зона — у проді за VPN/allowlist, з інтернету невидима.

## Як зайти (локально)

1. Підняти оточення: з `10-Projects/DBManager/code` → `docker compose up -d`.
2. Відкрити **http://localhost:8001** → перекине на вхід.
3. Логін: **`admin@dbmanager.local`** / **`admin`** (лише локальний dev; у проді змінюється).

Заходити саме через контейнерний порт `8001`, не через локальний `php artisan serve` (деталі — `documentation/Локальний запуск — порти й доступи.md`).

## Що вже вміє (план 2.1)

- **Грід «Значення»**: значення обраного сайта, згруповані за типами, з лічильниками.
- **Статуси failover** кольоровими плашками: `ok` / `на резерві` / `закріплено` / `вичерпано`.
- **Область дії** в рядку: «Група» або «✎ цього сайта» (перекриття).
- **Фільтри**: тип / гео / статус / пошук за ключем.
- **Перемикач сайта** у хлібних крихтах (за групами).
- Бічна навігація-каркас (Дашборд / Значення / Сайти і групи / Інциденти / Аудит / Доступи).

## Панель слота (план 2.2)

Клік на рядок-телефон у гріді відкриває праву панель, де можна **редагувати** слот:
- **Закріпити номер** (pin «Показувати цей») / зняти закріплення — вимикає авто-перемикання.
- **Режим повернення**: Авто / Sticky.
- **Політика вичерпання**: прибрати вивід / показувати останній / аварійний.
- **Прив'язані месенджери**: увімкнути/вимкнути.
- **Зберегти** → публікація уражених сайтів у DataBridge (зміни одразу пишуться в аудит).

## Керування значеннями (план 2.3)

Кнопка **«+ Додати значення»** і клік на не-телефонному рядку відкривають редактор:
- **Додати / редагувати / видалити** значення (тип, ключ, вміст).
- **Область дії** — головне для «одного vs кількох сайтів»:
  - **Уся група** — одна зміна оновлює **всі сайти групи** одразу (так робиться «для кількох сайтів»);
  - **Цей сайт** — значення лише для одного сайта (або перекриття групового).
- **«Перекрити для цього сайта»** на груповому рядку — створює власну версію сайта (бейдж «✎ цього сайта»), що виграє над груповою.
- Після збереження — автоматична публікація **всіх уражених сайтів** у DataBridge + запис в аудит.

## Телефонні номери у слоті (план 2.4)

Телефон додається як значення типу **«Телефон»** («+ Додати значення» → тип Телефон): створюється слот, і одразу відкривається його панель. У панелі слота керуєш ланцюгом номерів:
- **+ додати резерв** — ввести номер у форматі `+380XXXXXXXXX`;
- **✕** — видалити номер із ланцюга (спільний номер у БД лишається — він може бути в інших слотах);
- **↑ / ↓** — змінити пріоритет (хто основний, хто резерв);
- зміни одразу перераховують показуваний номер (failover); **«Зберегти»** публікує в bridge.

## Сайти і групи (план 2.8)

Екран **«Сайти і групи»** (`/admin/sites`) — точка входу в систему: звідси заводять сайти й заходять у їхні дані. Список зліва, права панель налаштувань — як на екрані «Доступи».

- **Групи й сайти**: створити / перейменувати / редагувати; сайт має домен, назву, гео-підказку країни і (необов'язково) групу.
- **Архів замість видалення**: «Заархівувати» ховає сайт або групу, але **дані лишаються**; архів групи ховає й усі її сайти. Перемикач «Показати архів» + кнопка «Відновити» повертають усе назад. Домен заархівованого сайта вважається зайнятим — щоб ним скористатися знову, сайт треба відновити.
- **Токени сайта** (у панелі сайта): **Видати / Відкликати / Ротувати**. Сирий токен показується **один раз** — скопіюйте одразу, далі його не видно. Це ключ, яким сайт-плагін забирає дані з DataBridge.
- **Стан зв'язку** в панелі: чинний токен / коли сайт востаннє звертався / яку версію публікації отримав.
- **«Керувати даними»** у рядку сайта → відкриває грід «Значення» вже прив'язаний до цього сайта (головний робочий маршрут). Архівований сайт у дані не відкривається.
- **Хто може**: усі зміни тут — лише для **суперадміна**; кожна дія (групи, сайти, токени) пишеться в аудит.

## Масові операції

Екран **«Масові операції»** (`/admin/bulk`) потрібен, коли одне значення треба змінити на багатьох сайтах одразу.

- **Область**: усі доступні сайти, конкретна група, вручну вибрані сайти або сайт разом із дочірніми.
- **Фільтри**: тип даних, стан, geo, пошук по `key/content/url`, номер або фрагмент номера.
- **Превʼю**: таблиця показує, які сайти, ключі, geo, стани й поточні значення будуть зачеплені; якщо цілей більше 80, екран показує перші 80 і попереджає про це.
- **Дії**: знайти/замінити в `key/content`, встановити `content.value`, змінити geo, показати/приховати слот, замінити номер, перевести номер у `active/down`.
- **Безпека**: заміна номера не запускається без фільтра номера; перед застосуванням є browser-confirm, після зміни створюється batch-id в аудиті.
- **Публікація**: прапорець **«Публікувати сайти після зміни»** увімкнений за замовчуванням; якщо його вимкнути, зміни лишаться в Core без пушу в DataBridge.

## Чого ще немає (наступні плани)

- Відкат batch-операції з UI, збережені пресети масових змін, drag-drop пріоритетів номерів.
- Дашборд, дзвіночок інцидентів, аудит-стрічка з відкатом, ролі/права, імпорт CSV/XLSX — плани 2.x.
